# 实战：Spring Boot 项目搭建与 Bean 装配 — 操作步骤详解

> 面向后端新手。这篇文档走两条线：**① 先把 Spring Boot 项目跑起来（最小可运行版本）；② 再理解业务类之间的依赖，以及 Spring 提供的三种装配方式（注解 / XML / Java Config）**。代码部分来自 `Java-P1` 项目和课程截图。

## 开篇：一图看懂全局

### 这节课干了什么（两条主线）

```
主线 ① — 先把项目跑起来
pom.xml → Application.java → HelloController.java
→ 浏览器访问 http://localhost:8080/ 看到字符串 ✓

主线 ② — 再理解 Bean 装配
引入业务类：User → UserService → OrderService
     │
     ├─ 注解方式：@Inject（让容器注入依赖）
     ├─ XML 方式：applicationContext.xml（<bean> 标签登记）
     └─ Java Config 方式：@Configuration + @Bean（Java 代码登记）
```

### 业务类依赖关系图

```
HelloController
    │ 依赖（@Inject）
    ▼
OrderService
    │ 依赖（@Inject）
    ▼
UserService
    │ 调用
    ▼
User（数据对象）
```

### 三种装配方式对比

| 方式 | 在哪声明 Bean | 优点 | 缺点 |
|------|-------------|------|------|
| **注解**（`@Component`/`@Service` + `@Inject`） | 直接在业务类上打标注 | 最简洁，最常用 | 需要额外 jar（`javax.inject`） |
| **XML**（`applicationContext.xml`） | XML 配置文件 | 清晰分离配置与代码 | 冗长，不直观 |
| **Java Config**（`@Configuration` + `@Bean`） | 配置类里的方法 | 类型安全，可编程 | 显式，但不如注解简洁 |

### 技术关键词速查

| 关键词 | 一句话 |
|--------|--------|
| `@SpringBootApplication` | 启动类注解，触发扫描 + 自动配置 + 容器创建 |
| `@RestController` | 处理 Web 请求，返回值直接作为 HTTP body |
| `@GetMapping("/")`  | 把 GET / 请求映射到方法 |
| `@Inject` | 告诉容器：创建这个类时，把依赖传进来 |
| `@Component` / `@Service` | 声明这个类是 Bean（让容器扫描到并注册） |
| `@Configuration` | 声明这是配置类（可以包含 `@Bean` 方法） |
| `@Bean` | 方法返回的对象，注册进容器成为 Bean |
| `@ImportResource` | 把 XML 配置导入 Spring Boot 上下文 |
| `CommandLineRunner` | 容器启动后执行一段代码的钩子（常用于演示/调试） |

## 第一部分：实战步骤（可直接照做）

### 步骤 1：写 `pom.xml`

```xml
<?xml version="1.0" encoding="UTF-8"?>
<project xmlns="http://maven.apache.org/POM/4.0.0"
         xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
         xsi:schemaLocation="http://maven.apache.org/POM/4.0.0 http://maven.apache.org/xsd/maven-4.0.0.xsd">
    <modelVersion>4.0.0</modelVersion>

    <groupId>com.ygm</groupId>
    <artifactId>java-p1</artifactId>
    <version>0.0.1-SNAPSHOT</version>

    <parent>
        <groupId>org.springframework.boot</groupId>
        <artifactId>spring-boot-starter-parent</artifactId>
        <version>2.0.5.RELEASE</version>
    </parent>

    <dependencies>
        <!-- Web 能力：Spring MVC + 内嵌 Tomcat -->
        <dependency>
            <groupId>org.springframework.boot</groupId>
            <artifactId>spring-boot-starter-web</artifactId>
        </dependency>

        <!-- @Inject 注解需要这个 jar -->
        <dependency>
            <groupId>javax.inject</groupId>
            <artifactId>javax.inject</artifactId>
            <version>1</version>
        </dependency>
    </dependencies>

    <repositories>
        <repository>
            <id>alimaven</id>
            <name>aliyun maven</name>
            <url>http://maven.aliyun.com/nexus/content/groups/public/</url>
        </repository>
    </repositories>

    <properties>
        <java.version>1.8</java.version>
    </properties>

    <build>
        <plugins>
            <plugin>
                <groupId>org.springframework.boot</groupId>
                <artifactId>spring-boot-maven-plugin</artifactId>
            </plugin>
        </plugins>
    </build>
</project>
```

> **注意：** `javax.inject` 只是 `@Inject` 注解的来源，不是 Spring 必须的。后面课程如果改用 `@Autowired`，可以去掉这个依赖。

### 步骤 2：写启动类 `Application.java`

文件路径：`src/main/java/hello/Application.java`

```java
package hello;

import org.springframework.boot.SpringApplication;
import org.springframework.boot.autoconfigure.SpringBootApplication;

@SpringBootApplication
public class Application {
    public static void main(String[] args) {
        SpringApplication.run(Application.class, args);
    }
}
```

**启动类的作用：**
- `main()` → JVM 入口
- `SpringApplication.run(...)` → 创建容器、扫描 Bean、启动内嵌 Tomcat

> **注意：** 启动类放在根包 `hello/` 下，这样 `hello.service`、`hello.controller` 等子包都能被扫描到。

### 步骤 3：写第一个控制器 `HelloController.java`

文件路径：`src/main/java/hello/HelloController.java`

**最小版本（先能跑起来）：**

```java
package hello;

import org.springframework.web.bind.annotation.GetMapping;
import org.springframework.web.bind.annotation.RestController;

@RestController
public class HelloController {

    @GetMapping("/")
    public String index() {
        return "Greetings from Spring Boot!";
    }
}
```

运行后访问 `http://localhost:8080/` → 看到 `Greetings from Spring Boot!` ✓

### 步骤 4：引入业务类（理解依赖关系）

三个业务类：`User`、`UserService`、`OrderService`。

**先不加 Spring 注解，只看业务关系：**

#### `User.java`

```java
package hello.service;

public class User {
    private Integer id;
    private String name;

    public User(Integer id, String name) {
        this.id = id;
        this.name = name;
    }

    public Integer getId() { return id; }
    public void setId(Integer id) { this.id = id; }

    public String getName() { return name; }
    public void setName(String name) { this.name = name; }
}
```

#### `UserService.java`

```java
package hello.service;

public class UserService {
    public User getUserById(Integer id) {
        return new User(id, "name");  // 假数据，后面会改成真实查数据库
    }
}
```

#### `OrderService.java`

```java
package hello.service;

public class OrderService {
    private UserService userService;

    public OrderService(UserService userService) {
        this.userService = userService;  // 构造器接收依赖
    }

    public void placeOrder(Integer userId, String item) {
        userService.getUserById(userId);
    }
}
```

> **这组代码在说什么：**
> - `User`：只是装数据的普通对象
> - `UserService`：业务逻辑类，负责"查用户"
> - `OrderService`：依赖 `UserService`，说明业务类之间会互相协作
> - **OrderService 依赖 UserService**，体现的是 A 干活离不开 B

### 步骤 5：方式一 —— 注解装配（最常用）

用 `@Inject` + `@Component` / `@Service` 告诉容器"谁是 Bean"以及"谁依赖谁"。

**改造 `UserService.java`（加 `@Service` 声明为 Bean）：**

```java
package hello.service;

import org.springframework.stereotype.Service;

@Service   // 告诉容器：这个类是 Bean，由容器创建和管理
public class UserService {
    public User getUserById(Integer id) {
        return new User(id, "name");
    }
}
```

**改造 `OrderService.java`（加 `@Service` + `@Inject`）：**

```java
package hello.service;

import org.springframework.stereotype.Service;
import javax.inject.Inject;

@Service
public class OrderService {
    private UserService userService;

    @Inject  // 告诉容器：创建 OrderService 时，把 UserService 注入进来
    public OrderService(UserService userService) {
        this.userService = userService;
    }

    public void placeOrder(Integer userId, String item) {
        userService.getUserById(userId);
    }
}
```

**改造 `HelloController.java`（注入 `OrderService`）：**

```java
package hello;

import hello.service.OrderService;
import org.springframework.web.bind.annotation.GetMapping;
import org.springframework.web.bind.annotation.RestController;
import javax.inject.Inject;

@RestController
public class HelloController {
    private OrderService orderService;

    @Inject
    public HelloController(OrderService orderService) {
        this.orderService = orderService;
    }

    @GetMapping("/")
    public String index() {
        return "Greetings from Spring Boot!";
    }
}
```

> **核心要点：** `@Inject` 不是创建对象，而是声明"这里需要一个对象"。创建对象是容器的事，声明需要是你代码的事。

---

### 步骤 6：方式二 —— XML 配置装配

适合老项目或需要清晰分离配置与代码的场景。

**1. 创建 `applicationContext.xml`**

文件路径：`src/main/resources/applicationContext.xml`

```xml
<?xml version="1.0" encoding="UTF-8"?>
<beans xmlns="http://www.springframework.org/schema/beans"
       xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
       xsi:schemaLocation="
           http://www.springframework.org/schema/beans
           http://www.springframework.org/schema/beans/spring-beans.xsd">

    <bean id="userService" class="hello.service.UserService"/>
    <bean id="orderService" class="hello.service.OrderService"/>
</beans>
```

- `<bean>` = 向 Spring 登记一个对象
- `id` = Bean 的名字（也是注入时的标识）
- `class` = Bean 对应哪个 Java 类

**2. 写 `XmlConfiguration.java` 让 Spring Boot 读这份 XML**

```java
package hello.configuration;

import org.springframework.context.annotation.Configuration;
import org.springframework.context.annotation.ImportResource;

@Configuration
@ImportResource({"classpath*:applicationContext.xml"})
public class XmlConfiguration {
    // 空类，只作为桥梁：把 XML 里的 Bean 定义导入 Spring Boot 上下文
}
```

> **注意：** Spring Boot 默认不读 `applicationContext.xml`，必须用 `@ImportResource` 显式导入才能生效。

### 步骤 7：方式三 —— Java Config 装配

用 Java 方法返回 Bean 实例，比 XML 更类型安全，也更便于 IDE 检查。

```java
package hello.configuration;

import hello.service.OrderService;
import hello.service.UserService;
import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;

@Configuration
public class JavaConfiguration {

    @Bean
    public UserService userService() {
        return new UserService();
    }

    @Bean
    public OrderService orderService(UserService userService) {
        // Spring 会自动把容器里的 UserService 传进来
        return new OrderService(userService);
    }
}
```

> **核心要点：** `@Bean` 方法的**参数**，Spring 会按类型从容器里找对应 Bean 传进来。所以 `orderService(UserService userService)` 里的 `userService`，不是你手动传的，是 Spring 自动注入的。

## 第二部分：核心知识点（面试 / 工作常问）

### 知识点 1：`@Inject`、`@Autowired`、`@Resource` 怎么区分

| 注解 | 来源 | 默认匹配方式 | 工作中常见度 |
|------|------|------------|------------|
| `@Autowired` | Spring 自带 | 按**类型** | ★★★ 最常用 |
| `@Inject` | JSR-330 标准（`javax.inject`） | 按**类型** | ★★ 常见 |
| `@Resource` | JSR-250 标准（`javax.annotation`） | 按**名称**（默认） | ★ 较少 |

**对初学者来说，共同点更重要：三者都是"告诉容器帮我注入依赖"，先只记 `@Autowired` 即可。**

---

### 知识点 2：三种 Bean 声明方式的本质是一样的

```
告诉 Spring 容器："这个类/这个对象，你来管"

注解方式：  @Component / @Service / @Repository / @Controller  （在类上标注）
XML 方式：  <bean id="..." class="..." />  （在 XML 文件里登记）
Java Config：  @Configuration + @Bean  （在配置类里用方法返回对象）
```

三种方式本质都在做**同一件事**：让容器认识这个 Bean，知道谁依赖谁，启动时自动组装。

---

### 知识点 3：`@Configuration` 是什么意思

告诉 Spring：**"这个类不是普通业务类，而是专门给 Spring 提供配置的类。"**

类比：
- **XML 配置文件** = 告诉容器有哪些 Bean
- **`@Configuration` 类** = 用 Java 代码写的"配置文件"

所以 `@Configuration` 类里的 `@Bean` 方法，等价于 XML 里的 `<bean>` 标签。

---

### 知识点 4：`@Bean` 方法的参数为什么可以被自动注入

`@Bean` 方法不是普通方法，而是 Spring 用来**创建 Bean 的工厂方法**。

```java
@Bean
public OrderService orderService(UserService userService) {
    return new OrderService(userService);
}
```

执行流程：
1. Spring 看到这个方法，知道要创建 `OrderService`
2. 看到参数需要 `UserService`
3. 去容器里找 `UserService` Bean（必须已经创建好）
4. 把找到的 `UserService` 传进来
5. 执行 `return new OrderService(userService)` 得到 Bean 实例

等价于你手写：
```java
UserService us = new UserService();
OrderService os = new OrderService(us);
```

只是前半段"准备 UserService"由容器帮你做了。

---

### 知识点 5：`@Bean` 方法里的推荐写法 vs 不推荐写法

```java
// ✓ 推荐：参数写法，一眼看出依赖关系，依赖由容器管理
@Bean
public OrderService orderService(UserService userService) {
    return new OrderService(userService);
}

// ✓ 也可以：调用同类里的另一个 @Bean 方法，Spring 保证单例
@Bean
public OrderService orderService() {
    return new OrderService(userService());
}

// ✗ 不推荐：手动 new，看起来像又自己造了一个，不利于理解"依赖交给容器"
@Bean
public OrderService orderService() {
    return new OrderService(new UserService());
}
```

**建议：** 在 Java Config 里，如果一个 Bean 依赖另一个 Bean，优先把依赖写到方法参数里。

---

### 知识点 6：构造器注入 vs 字段注入 vs Setter 注入

| 注入方式 | 代码形式 | 推荐度 | 说明 |
|---------|---------|-------|------|
| **构造器注入**（推荐） | `@Inject public Service(Dep dep)` | ★★★ | 依赖不可为 null，不可变，便于测试 |
| **字段注入** | `@Autowired private Dep dep;` | ★★ | 简洁，但不便于单元测试（无法 mock） |
| **Setter 注入** | `@Autowired public void setDep(Dep dep)` | ★ | 依赖可选时用，不常见 |

**工作中：** Spring 官方推荐构造器注入，IDEA 对字段注入也会给出黄色提示建议改为构造器注入。

---

### 知识点 7：`CommandLineRunner` 是什么（了解即可）

```java
@Bean
public CommandLineRunner commandLineRunner(ApplicationContext ctx) {
    return args -> {
        System.out.println("容器里的 Beans：");
        Arrays.stream(ctx.getBeanDefinitionNames()).sorted().forEach(System.out::println);
    };
}
```

- Spring Boot 应用启动完成后，会执行实现了 `CommandLineRunner` 接口的 Bean
- 常用于：启动后打印调试信息、初始化数据、触发某些任务
- 这节课老师用它演示容器里创建了哪些 Bean

## 第三部分：学习 Q&A

### Q1：`@Service` 和 `@Component` 有什么区别？

**A：** 功能完全一样，语义不同。

| 注解 | 语义 | 适用场景 |
|------|------|---------|
| `@Component` | 通用组件 | 其他注解不合适时的通用选择 |
| `@Service` | 业务服务层 | Service 类（业务逻辑） |
| `@Repository` | 数据访问层 | DAO / Mapper 类 |
| `@Controller` | Web 控制层 | 处理 HTTP 请求的类 |

它们都能让 Spring 扫描到并注册成 Bean，区别只是**语义**（让代码更自描述），`@Service`/`@Repository`/`@Controller` 本质上都是 `@Component` 的特化。

---

### Q2：`@Bean` 和 `@Component` 都能注册 Bean，有什么区别？

**A：**

| | `@Component`（及其变体） | `@Bean` |
|--|----------------------|---------|
| **用在哪** | 类上 | 方法上（在 `@Configuration` 类里） |
| **谁的类** | 你自己写的类 | 可以是第三方库的类（你没法改源码加注解） |
| **控制力** | 低（由 Spring 自动调用构造器） | 高（你手动写 `new`，可以做复杂初始化） |

**常见场景：** 自己的业务类用 `@Service`/`@Component`；第三方库的类（比如某个 SDK 的 Client）用 `@Bean` 注册。

---

### Q3：Spring 怎么解决循环依赖？

**A：** 这是一个高频面试题。简版回答：

- Spring **单例 Bean 的构造器注入**不支持循环依赖（会报错）
- **字段注入 / Setter 注入**的单例 Bean，Spring 通过"三级缓存"机制（提前暴露半成品 Bean）解决循环依赖
- Spring Boot 2.6+ 默认禁止循环依赖（需要加 `spring.main.allow-circular-references=true` 才能开启）

**实际建议：** 出现循环依赖通常是设计问题（A 依赖 B，B 又依赖 A），应该重新思考分层结构，而不是靠框架绕过。

---

### Q4：注解、XML、Java Config 三种方式，工作中用哪种？

**A：** 现在工作中 **90% 的场景用注解方式**（`@Service`、`@Autowired`），原因是最简洁、最直观。

- **Java Config**：用于注册第三方库的 Bean，或需要编程式控制初始化逻辑的场景
- **XML**：基本只在维护老项目时才会见到，新项目不推荐

所以学习优先级：**注解 > Java Config > XML**。

---

### Q5：`@Bean` 方法里为什么 `userService()` 和 `UserService` 不会搞混？

**A：** Java 里，类名首字母大写，方法名首字母小写，两者在语法上是完全不同的东西：

- `UserService`：类型名（大写开头）
- `userService()`：方法名（小写开头）

```java
// 这里：
// - UserService（返回值类型）：是类名
// - userService()（方法名）：是方法，刚好和类名同名但首字母小写
@Bean
public UserService userService() {
    return new UserService();  // new UserService() 是调用 UserService 的构造器
}
```

`userService()` 也不是构造函数（构造函数名必须和类名完全一致，且没有返回值类型），它是一个普通方法。
