# Spring Boot 接口、Docker 部署与 Java 运行环境

> 面向后端新手。这节课的主线是：**一个 HTTP 请求怎么进 Spring Boot 代码、打成 JAR 后怎么跑进 Docker 容器对外提供服务、开发环境 JDK 和容器里 JVM 版本的关系**。

---

## 开篇：一图看懂全局

### 这节课干了什么（3 条线）

| 线 | 目标 | 最终效果 |
|----|------|----------|
| **第一条：HTTP 接口与参数** | `@RestController` + 路由注解 + `@RequestParam` | 浏览器访问 `/?user=张三` 返回 HTML，POST `/login` 接收表单参数 |
| **第二条：打包 + 容器化** | Maven 打 JAR，写 Dockerfile，`docker run` | 把 Spring Boot 应用跑进 Docker 容器，对外暴露端口 |

### 请求进入 Spring Boot 的链路图

```
浏览器 / curl
   │
   │  GET /?user=张三  或  POST /login
   ▼
Docker 容器（80 端口） → 端口映射 → 宿主机 80
   │
   │  docker run -p 80:8080 → 转发到容器内 8080
   ▼
Spring Boot（内嵌 Tomcat 监听 8080）
   │
   ▼
DispatcherServlet（统一入口，分发请求）
   │
   ├─ GET /  → HelloController.index()  → 返回 HTML 字符串
   └─ POST /login → HelloController.login() → 接收 username/password
```

### Java 三层版本关系图

```
开发机（JDK 11）
   │
   │  javac + Maven 编译
   │  pom.xml 设置 <java.version>1.8</java.version>
   ▼
.class 文件（字节码目标版本 = Java 8）← 这里才是关键
   │
   │  打成 FAT JAR
   ▼
Docker 容器（FROM java:8u111-jdk-alpine）
   │
   │  java -jar app.jar
   ▼
JVM 8 运行 → 字节码版本(8) ≤ JVM版本(8) → 可以运行 ✓
```

### 技术栈关键词速查

| 关键词 | 一句话 |
|--------|--------|
| `@RestController` | 声明该类处理 Web 请求，返回值直接作为 HTTP body |
| `@RequestMapping` | 把 URL 路径映射到 Java 方法（可指定 GET/POST） |
| `@RequestParam` | 从 URL 查询参数或表单里取值 |
| `resources/static/` | Spring Boot 自动托管静态资源的目录 |
| `DispatcherServlet` | Spring MVC 的统一请求分发器，所有请求先经过它 |
| `Dockerfile` | 描述怎么构建镜像的文件 |
| `EXPOSE 8080` | 声明容器内应用监听的端口 |
| `-p 80:8080` | 把宿主机 80 端口流量转发到容器内 8080 |
| `spring-boot-maven-plugin` | 把项目打成可执行 Fat JAR 的插件 |
| 字节码目标版本 | `.class` 文件能跑在哪个 JVM 上，由 `<java.version>` 控制 |

## 第一部分：实战步骤（可直接照做）

### 步骤 1：写 `HelloController`（包含 GET 和 POST 接口）

```java
package hello;

import org.springframework.web.bind.annotation.RequestMapping;
import org.springframework.web.bind.annotation.RequestMethod;
import org.springframework.web.bind.annotation.RequestParam;
import org.springframework.web.bind.annotation.RestController;

@RestController
public class HelloController {

    // GET / —— 接收查询参数 ?user=xxx，返回 HTML 字符串
    @RequestMapping("/")
    public String index(@RequestParam("user") String id) {
        return String.format("<h1>Hello, %s!</h1>", id);
    }

    // POST /login —— 接收表单参数 username 和 password
    @RequestMapping(value = "/login", method = RequestMethod.POST)
    public String login(
        @RequestParam("username") String username,
        @RequestParam("password") String password
    ) {
        System.out.println("username: " + username + ", password: " + password);
        return "";  // 真实项目这里会验证并返回 JSON
    }
}
```

> **注意点：**
> - `@RestController` = `@Controller` + `@ResponseBody`，方法返回值直接作为 HTTP 响应体
> - `@RequestParam("user")` 从 URL `?user=张三` 里取值；如果参数不传，默认会报 400 错误
> - POST `/login` 的参数来自表单 body，不是 URL 里的 `?`
> - 这里返回的是 HTML 字符串，不是 JSON，属于 SSR（服务端渲染）的简单形态

---

### 步骤 2：放静态资源

把前端文件（`index.html`、`.js`、图片等）放到：

```
src/main/resources/static/
```

Spring Boot 会自动托管这个目录下的文件：

- 访问 `http://localhost:8080/index.html` → 直接返回文件内容
- 访问 `http://localhost:8080/laowang.png` → 直接返回图片

> **注意：** 静态资源路径和 Controller 路径不能冲突。如果 Controller 已经映射了 `/`，访问根路径时走 Controller 而不走 `index.html`。

---

### 步骤 3：配置 `pom.xml` 打可执行 JAR

```xml
<properties>
    <java.version>1.8</java.version>   <!-- 控制字节码目标版本 -->
</properties>

<build>
    <plugins>
        <plugin>
            <groupId>org.springframework.boot</groupId>
            <artifactId>spring-boot-maven-plugin</artifactId>
        </plugin>
    </plugins>
</build>
```

打包命令：

```bash
mvn package
```

执行后 `target/` 目录下会出现一个可执行 JAR，文件名类似 `gs-spring-boot-0.1.0.jar`。

> **注意：** `<java.version>1.8</java.version>` 控制的是字节码目标版本（`.class` 文件），不是你电脑上安装的 JDK 版本。

---

### 步骤 4：写 `Dockerfile`

```dockerfile
FROM java:8u111-jdk-alpine

WORKDIR /app

COPY target/gs-spring-boot-0.1.0.jar .

EXPOSE 8080

CMD ["java", "-jar", "gs-spring-boot-0.1.0.jar"]
```

> **注意点：**
> - `FROM java:8u111-jdk-alpine`：容器里的 JVM 版本由此决定，必须 ≥ 字节码目标版本
> - `COPY target/...`：先本地 `mvn package` 打出 JAR，再 build 镜像，不然 COPY 会找不到文件
> - `EXPOSE 8080`：只是文档声明，真正把端口暴露出去靠 `docker run -p`

---

### 步骤 5：构建镜像并运行容器

```bash
# 构建镜像（在 Dockerfile 所在目录执行）
docker build -t my-spring-app .

# 运行容器，把宿主机 80 映射到容器内 8080
docker run -p 80:8080 my-spring-app
```

运行后：

- 浏览器访问 `http://localhost/?user=张三` → 看到 `Hello, 张三!`
- `-p 80:8080` 格式：宿主机端口 : 容器端口

## 第二部分：核心知识点（面试 / 工作常问）

### 知识点 1：`@RequestMapping` vs `@GetMapping` / `@PostMapping`

| 注解 | 适用范围 | 说明 |
|------|---------|------|
| `@RequestMapping("/")` | GET + POST + 所有方法 | 通用，不指定方法时接受所有 HTTP 方法 |
| `@RequestMapping(value="/login", method=RequestMethod.POST)` | 只接受 POST | 指定 HTTP 方法 |
| `@GetMapping("/")` | 只接受 GET | `@RequestMapping` + `method=GET` 的简写 |
| `@PostMapping("/login")` | 只接受 POST | `@RequestMapping` + `method=POST` 的简写 |

**工作中：** `@GetMapping` / `@PostMapping` 更常用，语义更清晰。

---

### 知识点 2：`@RequestParam` 的工作方式

```java
// GET /?user=张三
public String index(@RequestParam("user") String id)

// POST /login（表单 body）
public String login(@RequestParam("username") String username)
```

- 从 URL 查询参数（`?key=value`）或表单 body（Content-Type: form）里取值
- 默认必填，参数不传会报 400；加 `required = false` 可以设为选填

---

### 知识点 3：`DispatcherServlet` 是什么

- Spring MVC 的**核心入口**，所有 HTTP 请求先到它
- 它根据 `@RequestMapping` 等注解，找到该由哪个 Controller 方法处理
- 由 `spring-boot-starter-web` 自动配置，你不需要手动写

**记忆：** DispatcherServlet = 前台接待 → 分发到对应的部门（Controller）

---

### 知识点 4：Docker 端口映射 `-p` 的含义

```bash
docker run -p 80:8080 my-spring-app
#               ↑    ↑
#         宿主机端口  容器内端口
```

- 用户访问宿主机 `http://your-server-ip:80`
- Docker 把流量转发到容器内的 `8080`
- Spring Boot 在容器内监听 `8080`

**为什么这样做：** HTTP 默认端口是 80（浏览器不用写端口），但 Spring Boot 默认 8080；通过映射把两者连起来，对外暴露 80。

---

### 知识点 5：JDK / 字节码目标版本 / 容器 JVM —— 三件事（面试高频）

这是这节课最重要的概念，也是面试时被问到"为什么用 JDK 17 开发，但部署环境是 Java 8"时的答案。

| 概念 | 含义 | 举例 |
|------|------|------|
| **开发 JDK** | 你本机 `javac`、Maven、IDE 使用的 JDK 版本 | 本机装 JDK 11 |
| **字节码目标版本** | `.class` 文件的 major version，决定最低能跑在什么 JVM | `pom.xml` 设 `java.version=1.8`，就是 Java 8 字节码 |
| **容器 JVM** | Docker 镜像里实际跑 JAR 的 Java 版本 | `FROM java:8u111-jdk-alpine` |

**核心规则：** 容器 JVM 版本 ≥ 字节码目标版本 即可运行，开发 JDK 版本无限制（可以更高）。

**典型场景：**

```
本机 JDK 11 开发
→ pom.xml 设 <java.version>1.8</java.version>
→ 打出的 .class 是 Java 8 字节码
→ Docker 里 JVM 8 可以跑 ✓

本机 JDK 17 开发，但 pom.xml 没配置 java.version
→ 默认按 JDK 17 打出 Java 17 字节码
→ Docker 里 JVM 8 不能跑 ✗（版本太低）
→ 报 UnsupportedClassVersionError
```

---

### 知识点 6：IDEA 里 `Project SDK` 和 `Language level` 分别控制什么

| 设置 | 控制什么 | 影响范围 |
|------|---------|---------|
| **Project SDK = 11** | IDE 用哪个 JDK 编译和运行测试 | IDE 内部 |
| **Language level = 8** | 源码只能用 Java 8 及以前的语法 | 代码写法 |
| **`pom.xml` 的 `java.version`** | 打出的 `.class` 字节码版本 | 真正决定能跑在哪个 JVM 上 |

**注意：** Language level = 8 不代表打出的 JAR 就是 Java 8 字节码，真正控制字节码版本的是 `pom.xml` 的 `maven-compiler-plugin` 配置。IDEA 的 Language level 主要管"IDE 里的语法检查"。

## 第三部分：学习 Q&A

### Q1：`@RestController` 和 `@Controller` 有什么区别？

**A：** `@RestController` = `@Controller` + `@ResponseBody`

- `@Controller`：返回视图名（Spring 去找 HTML 模板），想返回 JSON/字符串需要每个方法加 `@ResponseBody`
- `@RestController`：返回值直接作为 HTTP body，不找模板，适合写 REST API

---

### Q2：`@RequestParam` 和 `@PathVariable` 有什么区别？

**A：** 取值来源不同。

```java
// @RequestParam：从 ?key=value 里取
// 访问：GET /user?id=1
@GetMapping("/user")
public String getUser(@RequestParam("id") Integer id)

// @PathVariable：从 URL 路径段里取
// 访问：GET /user/1
@GetMapping("/user/{id}")
public String getUser(@PathVariable("id") Integer id)
```

---

### Q3：`EXPOSE 8080` 写了就能对外访问吗？

**A：** 不能。`EXPOSE` 只是镜像文档说明（告诉别人这个镜像监听哪个端口），并不真的开放端口。

真正对外暴露端口，必须在 `docker run` 时加 `-p`：

```bash
docker run -p 80:8080 my-spring-app  # 这才真的开放
```

---

### Q4：如果本机 JDK 11 打出的 JAR，在 Docker JVM 8 里跑报 `UnsupportedClassVersionError`，怎么解决？

**A：** 在 `pom.xml` 里明确设置字节码目标版本为 8：

```xml
<properties>
    <java.version>1.8</java.version>
</properties>
```

或者更精确地配置 `maven-compiler-plugin`：

```xml
<plugin>
    <groupId>org.apache.maven.plugins</groupId>
    <artifactId>maven-compiler-plugin</artifactId>
    <configuration>
        <release>8</release>
    </configuration>
</plugin>
```

这样无论你本机 JDK 是什么版本，打出的 `.class` 文件都是 Java 8 字节码，可以在 JVM 8 上运行。

---

### Q5：静态资源和 Controller 路径冲突时，哪个优先？

**A：** Controller 优先。

如果 Controller 里有 `@RequestMapping("/")`，访问 `/` 时会走 Controller 方法，不会返回 `static/index.html`。

如果想返回 `index.html`，去掉 Controller 里对 `/` 的映射，或者改成其他路径。
