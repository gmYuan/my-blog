# 实战：从零开始创建 Spring Boot 项目

> 面向后端新手。这节课的主线是：**有哪些方式起一个 Spring Boot 工程 → 手搓目录与 `pom.xml` → Maven 全局/项目配置（镜像与依赖）→ 用官方示例写出第一个 Web 控制器**。与「Spring 原理（IoC/DI）」是另一条线：这里偏**工程怎么搭、构建工具怎么配、第一个 HTTP 接口怎么声明**。

---

### 这节课的主线

```
创建项目（4 种方式选一种）
        │

```

### 技术栈关键词速查

| 关键词 | 一句话 |
|--------|--------|
| `spring-boot-starter-parent` | 统一依赖版本管理，少写重复 `<version>` |
| `spring-boot-starter-web` | 拉进 Spring MVC + 内嵌 Tomcat，才能写 Controller |
| `spring-boot-maven-plugin` | `mvn package` 打出可执行 Fat JAR |
| `@SpringBootApplication` | 启动类标注，触发组件扫描 + 容器创建 + Tomcat 启动 |
| `SpringApplication.run(...)` | 真正把 Spring Boot 应用启动起来 |
| `@RestController` | 声明该类处理 HTTP 请求，返回值直接作为 body |
| `@RequestMapping("/"`) | 把 `/` 路径映射到该方法 |
| `settings.xml` | Maven 全局配置（常用来配镜像），所有项目都生效 |
| `pom.xml` | 当前项目配置（依赖、构建、仓库等） |
| `DispatcherServlet` | Spring MVC 的统一请求分发器 |

### 项目最小成品结构

```
springboot-demo/
├── pom.xml
└── src/
    └── main/
        └── java/
            └── hello/
                ├── Application.java       ← 启动类
                └── HelloController.java   ← 第一个控制器
```

## 第一部分：实战步骤（可直接照做）

### 步骤 1：创建项目目录

```bash
mkdir springboot-demo
cd springboot-demo
mkdir -p src/main/java/hello
```

> **为什么是 `src/main/java`：** Maven 约定的主源码根目录，Maven 和 IDEA 会自动识别。Spring Boot 打包插件也依赖这个布局。

验证：

```bash
tree .
# springboot-demo/
# └── src/
#     └── main/
#         └── java/
#             └── hello/
```

**4 种创建项目的方式（选一种即可）：**

| 方式 | 适合谁 |
|------|--------|
| **从 0 开始建目录**（本文） | 理解项目每一块是怎么来的，适合学习 |
| **`git clone` 模板** | 从团队模板或开源骨架开始，改坐标与包名 |
| **IDEA + Spring Initializr** | 图形界面勾选依赖，自动生成，最省时间 |
| **Maven archetype** | 命令行用原型生成骨架 |

### 步骤 2：写 `pom.xml`

在项目根目录（`springboot-demo/`）创建 `pom.xml`：

```xml
<?xml version="1.0" encoding="UTF-8"?>
<project xmlns="http://maven.apache.org/POM/4.0.0"
         xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
         xsi:schemaLocation="http://maven.apache.org/POM/4.0.0
         http://maven.apache.org/xsd/maven-4.0.0.xsd">
    <modelVersion>4.0.0</modelVersion>

    <!-- 项目身份证 -->
    <groupId>com.example</groupId>
    <artifactId>springboot-demo</artifactId>
    <version>0.0.1-SNAPSHOT</version>

    <!-- 继承 Spring Boot 父 POM：统一依赖版本 + 默认配置 -->
    <parent>
        <groupId>org.springframework.boot</groupId>
        <artifactId>spring-boot-starter-parent</artifactId>
        <version>2.0.5.RELEASE</version>
    </parent>

    <!-- 依赖：Web 能力（包含 Spring MVC + 内嵌 Tomcat） -->
    <dependencies>
        <dependency>
            <groupId>org.springframework.boot</groupId>
            <artifactId>spring-boot-starter-web</artifactId>
            <!-- 不需要写 version，父 POM 已经统一管理 -->
        </dependency>
    </dependencies>

    <!-- 项目级仓库（可选：加速国内下载） -->
    <repositories>
        <repository>
            <id>alimaven</id>
            <name>aliyun maven</name>
            <url>http://maven.aliyun.com/nexus/content/groups/public/</url>
        </repository>
    </repositories>

    <!-- 字节码目标版本 -->
    <properties>
        <java.version>1.8</java.version>
    </properties>

    <!-- 打包插件：打出可执行 Fat JAR -->
    <build>
        <plugins>
            <plugin>
                <groupId>org.springframework.boot</groupId>
                <artifactId>spring-boot-maven-plugin</artifactId>
            </plugin>
        </plugins>
    </build>
</project>
```

**这份 `pom.xml` 在告诉 Maven 4 件事：**

| 项 | 含义 |
|----|------|
| `groupId` / `artifactId` / `version` | 项目的"身份证" |
| `parent` | 给你一套 Spring Boot 推荐好的版本和默认配置 |
| `dependencies` | 这个项目要用哪些外部库 |
| `plugin` | 构建时额外做什么（这里是打 Boot JAR） |

### 步骤 3：写启动类 `Application.java`

文件路径：`src/main/java/hello/Application.java`

```java
package hello;

import org.springframework.boot.SpringApplication;
import org.springframework.boot.autoconfigure.SpringBootApplication;

@SpringBootApplication
public class Application {
    public static void main(String[] args) {
        SpringApplication.run(Application.class, args);
    }
}
```

**这段代码做了什么：**

```
JVM 执行 main()
    │
    ▼
SpringApplication.run(Application.class, args)
    │
    ├─ 读取 @SpringBootApplication 触发：
    │    ├─ 组件扫描（从 Application 所在包向下扫）
    │    ├─ 自动配置（根据 classpath 里的依赖自动配置）
    │    └─ 创建 Spring 容器
    │
    └─ 启动内嵌 Tomcat，监听 8080 端口
```

> **注意点：**
> - `Application.class`：告诉 Spring Boot 从哪个类开始启动（决定扫描起点）
> - `args`：把命令行参数传给 Spring Boot（例如 `--server.port=8081`）
> - 启动类建议放在根包 `hello/` 下，这样 `hello.service`、`hello.controller` 等子包都能被扫描到

---

### 步骤 4：写第一个控制器 `HelloController.java`

文件路径：`src/main/java/hello/HelloController.java`

```java
package hello;

import org.springframework.web.bind.annotation.GetMapping;
import org.springframework.web.bind.annotation.RestController;

@RestController
public class HelloController {

    @GetMapping("/")
    public String index() {
        return "Greetings from Spring Boot!";
    }
}
```

**这段代码做了什么：**

| 代码 | 含义 |
|------|------|
| `@RestController` | 这是一个处理 Web 请求的类，返回值直接作为 HTTP 响应 body |
| `@GetMapping("/")` | 收到 GET `/` 请求时，执行 `index()` 方法 |
| `return "Greetings..."` | 浏览器看到的内容 |

> **`@GetMapping` vs `@RequestMapping`：** `@GetMapping("/")` 是 `@RequestMapping(value="/", method=GET)` 的简写，语义更清晰，工作中优先用。

### 步骤 6：配置 Maven 全局镜像（可选，加速国内下载）

编辑 `~/.m2/settings.xml`（没有则新建）：

```xml
<settings>
    <mirrors>
        <mirror>
            <id>alimaven</id>
            <name>aliyun maven</name>
            <url>http://maven.aliyun.com/nexus/content/groups/public/</url>
            <mirrorOf>central</mirrorOf>
        </mirror>
    </mirrors>
</settings>
```

> **注意：** `settings.xml` 是全局配置（所有项目都走阿里云镜像下载）；`pom.xml` 里的 `<repositories>` 是项目级配置（只对这个项目生效）。两者不冲突，可以配合使用。

## 第二部分：核心知识点（面试 / 工作常问）

### 知识点 1：`spring-boot-starter-parent` 的作用

继承它时，父 POM 里有一份 `dependencyManagement`（可理解成"版本清单"）：Spring 生态里常用库的**兼容版本**已经写好了。

于是你在 `<dependencies>` 里写 `spring-boot-starter-web` 等依赖时，**往往不必再写 `<version>`** —— Maven 会按父 POM 的清单自动带上版本。

若没有这份 parent：每个 jar 都手写版本，容易出现不同依赖引了不同版本同一库的冲突问题。

---

### 知识点 2：Spring MVC 是什么？和 Spring Boot 什么关系？

| 层次 | 典型内容 |
|------|---------|
| **Spring Framework** | IoC 容器、**Spring MVC**、`DispatcherServlet`、`@Controller` 等 Web 编程模型 |
| **Spring Boot** | Starter（如 `spring-boot-starter-web`）、自动配置、内嵌服务器、可执行 JAR —— 在 Framework 之上"约定大于配置"，把 MVC + Tomcat 等开箱即用 |

**一句话：** MVC 和 `DispatcherServlet` 是 Spring Framework 的 Web 核心；Spring Boot 负责把它们和你的项目"接好线"并简化配置。

---

### 知识点 3：`DispatcherServlet` 是什么

- **是什么：** Spring MVC 的**前端控制器**（Front Controller），所有 Web 请求的统一入口
- **从哪来：** 通过 `spring-boot-starter-web` → `spring-webmvc` 依赖链带进来的
- **做什么：** HTTP 请求到达 → `DispatcherServlet` 根据 URL 映射表找到对应 Controller 方法 → 调用方法 → 把返回值写成响应体发回浏览器

---

### 知识点 4：`settings.xml` vs `pom.xml` 分工

| 文件 | 层级 | 常用来干什么 |
|------|------|------------|
| `~/.m2/settings.xml` | 用户全局 | 配 mirror（如阿里云镜像）：所有项目拉 jar 时走哪条"高速路"；也可配私服账号 |
| 项目根目录 `pom.xml` | 当前工程 | 写 `groupId` / `artifactId` / `version`、`<dependencies>`、`<repositories>`、构建插件 |

**防混：** `pom.xml` 里写的"Maven 依赖（jar 从哪来）"和"Spring 容器的 Bean 依赖（谁注入谁）"虽然都叫"依赖"，但是**完全不同的两层概念**。

---

### 知识点 5：`@SpringBootApplication` 做了什么（三合一）

`@SpringBootApplication` = `@SpringBootConfiguration` + `@EnableAutoConfiguration` + `@ComponentScan`

| 组成 | 作用 |
|------|------|
| `@SpringBootConfiguration` | 声明这是一个配置类 |
| `@EnableAutoConfiguration` | 根据 classpath 里的依赖自动配置（有 `spring-webmvc` 就自动配 MVC） |
| `@ComponentScan` | 从当前包开始扫描，找所有 `@Component`、`@Service`、`@Controller` 等 |

## 第三部分：学习 Q&A

### Q1：为什么启动类要放在根包下面，而不是放在子包里？

**A：** 因为 `@SpringBootApplication` 里的 `@ComponentScan` 默认从**启动类所在包**向下扫描子包。

如果启动类放在 `hello` 包，那么 `hello.service`、`hello.controller`、`hello.mapper` 等子包都能被扫到。

如果放在 `hello.config` 子包里，那 `hello.service`、`hello.controller` 就不在扫描范围内，Bean 就注册不上。

---

### Q2：`mvn package` 和 `mvn spring-boot:run` 有什么区别？

**A：**

| 命令 | 作用 |
|------|------|
| `mvn spring-boot:run` | 直接在 Maven 里启动 Spring Boot 应用，不生成 JAR，开发调试用 |
| `mvn package` | 编译 + 打包成可执行 JAR，然后用 `java -jar` 运行，模拟生产环境 |

开发阶段用 `mvn spring-boot:run`（或 IDEA 直接 Run），部署时用 `mvn package` + `java -jar`。

---

### Q3：项目里也配了 `<repository>`（阿里云），全局 `settings.xml` 也配了镜像，会冲突吗？

**A：** 不会冲突，两者配合使用：

- `settings.xml` 里的 `mirror` 是**拦截**所有去 `central` 的请求，重定向到阿里云
- `pom.xml` 里的 `<repository>` 是**额外声明**这个项目还可以从哪里拉包

通常只配 `settings.xml` 就够了，`pom.xml` 里的 `<repository>` 是可选的（用于特殊库或私服）。

---

### Q4：`spring-boot-starter-web` 里包含了什么？

**A：** 主要包含（间接依赖）：

| 包含的库 | 作用 |
|---------|------|
| `spring-webmvc` | Spring MVC 核心，包含 `DispatcherServlet`、`@Controller` 等 |
| `spring-boot-starter-tomcat` | 内嵌 Tomcat |
| `jackson-databind` | JSON 序列化/反序列化（返回 Java 对象会自动变成 JSON） |
| `spring-boot-starter` | Spring Boot 核心（自动配置、日志等） |

所以加了这一个 starter，Web 开发所需的全套就都有了。

---

### Q5：如果不继承 `spring-boot-starter-parent`，还能用 Spring Boot 吗？

**A：** 可以，但需要手动管理版本。一种常见做法是用 `dependencyManagement` + `import`：

```xml
<dependencyManagement>
    <dependencies>
        <dependency>
            <groupId>org.springframework.boot</groupId>
            <artifactId>spring-boot-dependencies</artifactId>
            <version>2.0.5.RELEASE</version>
            <type>pom</type>
            <scope>import</scope>
        </dependency>
    </dependencies>
</dependencyManagement>
```

这样可以享受 Spring Boot 的版本管理，但不必继承父 POM（适合公司已有自己的父 POM 的场景）。
