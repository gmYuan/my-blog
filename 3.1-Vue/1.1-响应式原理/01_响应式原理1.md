## 2 initData(vm: Component)

1 内部流程: <br/>

S1 getDataVal: 需要根据data的类型获取到 data和vm._data值
  - 理论上经过mergeOptions函数处理后，vm.$options.data的返回值一定是一个函数；但是在 mergeOptions-- beforeCreate-- initData流程中，用户还是可以在beforeCreate()阶段修改vm.$options.data值，所以此处需要进行保底的类型判断
  - getDate内会调用 pushTarget/popTarget [PS1]

S2 checkDataIsPlainObj: 确保data是一个纯对象
  - 只有data是纯对象类型时，才能有Obj.definePty属性，以实现响应式; 

S3 checkKeyHasDefined: 遍历对 data中的keyX进行同名校验
  - 防止data中的key和 props/methods中的key冲突，导致覆盖;
  - key定义的优先级值是 props > data > methods

S4 proxy_Data: 代理vm._data属性
  - 为了可以使用vm.key1 来快捷访问 vm._data.key1的值;
  - 设置代理前先检查了key不是保留key，因为保留key是在Vue内部使用的，没必要暴露给用户

S5 observeData: 把data转化为响应式数据
  - 从而实现 当data.keyX值发生变化时，所有读取了data.keyX的副作用函数都会自动执行
  - vm._data被标志为了rootData，从而用于识别针对rootData的特殊处理

[PS1] pushTarget/popTarget
  - 调用是为了防止 使用props数据初始化data数据时，收集冗余的依赖
  - 具体分析，需要见下篇文章分析

--------------------
2 代码实现
```ts
function initData(vm: Component) {
	//S1 需要根据data的类型获取到 data和vm._data值
  let data = vm.$options.data;
  data = vm._data = typeof data === "function" ? getData(data, vm) : data || {};
  //S2 确保data是一个纯对象
  if (!isPlainObject(data)) {
    data = {};
    process.env.NODE_ENV !== "production" &&
      warn(
        "data functions should return an object:\n" +
          "https://vuejs.org/v2/guide/components.html#data-Must-Be-a-Function",
        vm
      );
  }
  //S3 遍历对 data中的keyX进行同名校验
  const keys = Object.keys(data);
  const props = vm.$options.props;
  const methods = vm.$options.methods;
  let i = keys.length;
  while (i--) {
    const key = keys[i];
    if (process.env.NODE_ENV !== "production") {
      if (methods && hasOwn(methods, key)) {
        warn(
          `Method "${key}" has already been defined as a data property.`,
          vm
        );
      }
    }
    if (props && hasOwn(props, key)) {
      process.env.NODE_ENV !== "production" &&
        warn(
          `The data property "${key}" is already declared as a prop. ` +
            `Use prop default value instead.`,
          vm
        );
		//S4 用vm.key来代理vm._data.key: 
    } else if (!isReserved(key)) {
      proxy(vm, `_data`, key);
    }
  }
  //S5 把data转化为响应式数据
  observe(data, true /* asRootData */);
}

// getData
export function getData (data: Function, vm: Component): any {
  // #7573 disable dep collection when invoking data getters
  pushTarget()
  try {
    return data.call(vm, vm)
  } catch (e) {
    handleError(e, vm, `data()`)
    return {}
  } finally {
    popTarget()
  }
}

// proxy
const sharedPropertyDefinition = {
  enumerable: true,
  configurable: true,
  get: noop,
  set: noop
}
export function proxy (target: Object, sourceKey: string, key: string) {
  sharedPropertyDefinition.get = function proxyGetter () {
    return this[sourceKey][key]
  }
  sharedPropertyDefinition.set = function proxySetter (val) {
    this[sourceKey][key] = val
  }
  Object.defineProperty(target, key, sharedPropertyDefinition)
}
```

## 3 observe(value: any, asRootData: ?boolean):  

1 内部流程: <br/>

S1 setReturn: 设置递归中止条件
  - 从下文内容可以知道，observe其实是递归函数（在内部的子调用栈里调用了observe），所以需要设置递归中止条件
  - 中止条件是 当value是非对象类型，也就是说当value是基本数据类型时，它无法被观测，自然也不会有value.__ob__属性

S2 getOb: 根据value是否被监测过，读取/创建ob对象实例
  - 获取到 之前创建过的/新创建的 ob实例，从而 避免重复观测一个数据对象;
  - ob实例对象最后 会作为observe函数的返回值，被用于指向 后续的childOb

> ob对象的作用: 1.用作childOb，存储嵌套关系的依赖(读父子也存); 2. 用于存储那些无法走setter观测触发的依赖，如新增未定义的对象属性/调用数组变异方法等

S3 更新根data数据的 vmcount值
  - 下文会介绍，只有rootData的vmcount值为1，非rootData的vmcount值都为0，从而区分了rootData 和 非rootData
  - 至于为什么要区分rootData 和 非rootData，是因为rootData有一些特殊逻辑需要处理（比如下文介绍的Vue.set/delete）

--------------------------
2 代码实现
```ts
export function observe (value: any, asRootData: ?boolean): Observer | void {
  //S1 设置递归中止条件
  if (!isObject(value) || value instanceof VNode) {
    return
  }
  //S2 根据value是否被监测过，读取/创建ob对象实例
  let ob: Observer | void
  if (hasOwn(value, '__ob__') && value.__ob__ instanceof Observer) {
    ob = value.__ob__
  } else if (
    shouldObserve &&
    !isServerRendering() &&
    (Array.isArray(value) || isPlainObject(value)) &&
    Object.isExtensible(value) &&
    !value._isVue
  ) {
    ob = new Observer(value)
  }
  //S3 更新根data数据的 vmcount值
  if (asRootData && ob) {
    ob.vmCount++
  }
  return ob
}
```

## 4 Observer.construcrot(value: any)

1 内部流程: <br/>

S1 initObPty: 创建ob实例的属性
  - ob.dep: 存储某层对象/数组上，所有无法被setter触发的依赖
  - ob.value: 让ob和value关联上，以后通过ob 就能读到该ob那一层对应的value了
  - ob.vmCount: 非rootData的ob.vmCount值都是0

S2 defineData.__ob__: 创建value的__ob__属性
  - data._ob__.value === data，所以 它是一个循环引用 [PS1]

S3 handleArray/handleObject: 分别处理value 是数组/对象的情况
  - 在Vue2.X版本中，数组需要特殊实现响应式，具体原因见下文
  - 经过之前逻辑处理，一开始的rootData必然是一个对象类型，所以下文先讨论handleObject ==> this.waik()

## 6 defineReactive(obj, key, val, customSetter?:Fn, shallow?: Boolean)

1 内部流程: <br/>
S1 createSetterDep: 创建可以被setter触发的keyxDep
  - 为obj中的每个keyX创建dep，把该dep记做keyxDep
  - 当valX的值被改变 + 改变值方式会触发setter时，就会触发 KeyxDep里收集的依赖

S2 cacheGetterAndSetter: 缓存data__key 原本定义的getter/setter
  - 防止下面用 Obj.definePty重新定义的setter/getter，覆盖掉原有用户定义的getter/setter

S3 getVal: 根据data和key, 获取到对应val
  - 当不存在getter或者有setter + 只传递data和key时==> 才会触发getVal [PS1]

S4 getChildOb: 获取val那一层对象/数组，对应的ob实例
  - 因为data.keyX对应的valX，有可能还是一个数组/对象，所以需要把它也变成响应式数据，所以递归调用了observe(val)
  - 如果显式声明了是浅观测，那么就不会递归深度观测 子属性值
  - 如果没有触发getVal，此时val是undefined，也会隐式导致 深观测无效

S5 Obj.definePty.get: 正确返回属性值 + 使用dep闭包 收集依赖
  - 返回值vlaue会通过 原有getter调用/被setter更新后的val获取到
  - Dep.target会指向 当前读取了obj.keyX的 依赖

  - dep.depend(): 把当前依赖，收集到obj__key对应的dep里，记做keyDep。keyDep里的依赖，会在data__key对应的属性值被修改 + 修改值的方式可以被setter拦截到时，触发执行
  - childOb?.dep.depend(): 依赖读取了父对象，那么子对象也要收集该依赖 [PS2]
  - dependArray(value): 当value为数组时，子数组的每个成员对象也需要依赖收集  [PS3]

S6 Obj.definePty.set: 更新属性值 + 触发对应data__key的依赖
  - judgeValueChange: 对比新旧值是否相等，如果值没变化，则不需要触发更新
  - updateVal: 更新属性值
  - observe(newVal): 监测新的属性值
  - dep.notify(): 触发keyDep的依赖

> childOb的作用: 父子对象同步收集依赖（读父子也存）

[PS1] getVal的设计原因
  - 当属性原本就有getter函数时，就不会getVal/ 不深度观测这个属性，原因是：getter完全由用户定义，所以有可能引发不可预见行为，所以需要避免观测
  - 由上可知，当数据对象的某一个属性只有getter拦截器函数 + 没有setter拦截器函数时，此时该属性不会被深度观测。但是该属性在被 defineReactive函数处理后，该属性将被重新定义getter和 setter，且被深度观测了，这就存在一个矛盾：该属性原本不会被深度观测，但是重新赋值之后，新的值却被观测了
  - 这就是所谓的 定义响应式数据时行为的不一致，为了解决这个问题，采用的办法是当属性拥有setter时，即使拥有getter也要深度观测

[PS2] childOb?.dep.depend
  - 如果一个依赖efn1 读取了obj.key1，那么当obj.key1.child1值发生了变化时，自然也要让efn1自动执行，以获取最新的obj.key1.child1值
  - childObDep里的依赖，会在用 $set/Vue.set给数据对象 添加新属性时，触发执行==> 以弥补Vue2.X 直接添加新属性无法观测的问题

```ts
export function defineReactive (
  obj: Object,
  key: string,
  val: any,
  customSetter?: ?Function,
  shallow?: boolean
) {
  //S1 创建可以被setter触发的keyxDep
  const dep = new Dep()
  //S2 缓存data__key原本定义的getter/setter
  const property = Object.getOwnPropertyDescriptor(obj, key)
  if (property && property.configurable === false) {
    return
  }
  const getter = property && property.get
  const setter = property && property.set
  //S3 根据data和key, 获取到对应val
  if ((!getter || setter) && arguments.length === 2) {
    val = obj[key]
  }
  //S4 获取val那一层对象/数组，对应的ob实例
  let childOb = !shallow && observe(val)
  //S5+S6
  Object.defineProperty(obj, key, {
    enumerable: true,
    configurable: true,
    //S5 正确返回属性值 + 使用dep闭包 收集依赖
    get: function reactiveGetter () {
      // 获取最新返回值并返回
      const value = getter ? getter.call(obj) : val
      if (Dep.target) {
        // setterDep依赖收集
        dep.depend()
        if (childOb) {
          // 子层obDep依赖收集
          childOb.dep.depend()
          // 子数组成员对象依赖收集
          if (Array.isArray(value)) {
            dependArray(value)
          }
        }
      }
      return value
    },
    //S6 更新属性值 + 触发data__key存储的依赖
    set: function reactiveSetter (newVal) {
      // 对比新旧值是否相等 或者 新旧值都是NaN
      const value = getter ? getter.call(obj) : val
      /* eslint-disable no-self-compare */
      if (newVal === value || (newVal !== newVal && value !== value)) {
        return
      }
      // 一般用来打印辅助信息
      /* eslint-enable no-self-compare */
      if (process.env.NODE_ENV !== 'production' && customSetter) {
        customSetter()
      }
      // 更新属性值
      if (setter) {
        setter.call(obj, newVal)
      } else {
        val = newVal
      }
      // 监测新的属性值
      childOb = !shallow && observe(newVal)
      // 出发setterDep的依赖
      dep.notify()
    }
  })
}

// dependArray的实现
function dependArray (value: Array<any>) {
  for (let e, i = 0, l = value.length; i < l; i++) {
    e = value[i]
    e && e.__ob__ && e.__ob__.dep.depend()
    if (Array.isArray(e)) {
      // 继续递归子值
      dependArray(e)
    }
  }
}
```

## 5.2 handleArray()

1 内部流程: <br/>

S1 定义value.__ob__属性：同上对象类型的处理方式

S2 protoAugment(value, arrayMethods): 创建一个新的代理的 数组原型对象
  - 数组有很多可以改变值的原型方法(把这些方法称做变异方法)，调用他们不会触发getter/setter，所以需要特殊处理数组类型，从而可以拦截数组变异方法的调用
  - 变异方法有: push()/ pop()/ shift()/ unshift()/ splice()/ reverse()/ sort()等
  - 如何拦截数组变异方法: 
    - 获取原数组原型对象 arrayProto
    - 创建新的继承代理对象 arrayMethods
    - 在arrayMethods上定义同名的变异方法，并加入拦截处理逻辑
    - 拦截处理逻辑: 获取到数组实例上的ob对象 + 触发之前ob.dep上收集的依赖 + 观测新增的数组成员 + 返回变异方法执行结果
> 一句话总结: 代理数组原型对象 + 定义同名数组变异方法 + 加入拦截处理逻辑

S3 copyAugment(value, arrayMethods, arrayKeys)
  - why: 需要意兼容处理 浏览器不支持__proto__属性的情况
  - how: 直接在数组实例上定义 与变异方法同名的函数
> 一句话总结: 在数组实例上定义 变异方法同名函数

S4 ob.observeArray(value): 观测数组每个成员对象
  - why: 数组内部的成员，可能还是数组/对象，所以需要深度观测其内部每个成员
  - how: 递归观测数组成员
> 一句话总结: 递归观测每个数组成员

```ts
// Observer.js
import { arrayMethods } from './array'
const arrayKeys = Object.getOwnPropertyNames(arrayMethods)
export class Observer {
  value: any;
  dep: Dep;
  vmCount: number;
  constructor (value: any) {
    // 省略部分的代码见上
    def(value, '__ob__', this)
    if (Array.isArray(value)) {
      if (hasProto) {
        protoAugment(value, arrayMethods)
      } else {
        copyAugment(value, arrayMethods, arrayKeys)
      }
      this.observeArray(value)
    } else {
      this.walk(value)
    }
  }
  // 省略内容见上
  walk() {}
  observeArray (items: Array<any>) {
    for (let i = 0, l = items.length; i < l; i++) {
      observe(items[i])
    }
  }
}

// helpers
function protoAugment (target, src: Object) {
  target.__proto__ = src
}
function copyAugment (target: Object, src: Object, keys: Array<string>) {
  for (let i = 0, l = keys.length; i < l; i++) {
    const key = keys[i]
    def(target, key, src[key])
  }
}
// -----------------------------------------
// array.js
import { def } from '../util/index'

// 代理数组原型对象
const arrayProto = Array.prototype
export const arrayMethods = Object.create(arrayProto)
const methodsToPatch = [
  'push',
  'pop',
  'shift',
  'unshift',
  'splice',
  'sort',
  'reverse'
]
// 定义同名数组变异方法
methodsToPatch.forEach(function (method) {
  const original = arrayProto[method]
  def(arrayMethods, method, function mutator (...args) {
    const result = original.apply(this, args)
    // 加入拦截处理逻辑
    const ob = this.__ob__  // this指向的是数组实例
    let inserted
    switch (method) {
      case 'push':
      case 'unshift':
        inserted = args
        break
      case 'splice':
        inserted = args.slice(2)
        break
    }
    if (inserted) ob.observeArray(inserted)
    // 
    ob.dep.notify()
    return result
  })
})
```

## 二 Vue.set/vm.$set原理介绍

### 1 vm.$set/$delete 和 Vue.set/delete 关系

S1 本质上是一致的，都是引用的 src/core/observer/index.js文件中定义的 set函数和del函数


### 2 Vue.set/$set

1 内部流程: <br/>
S1 checkTargetType: 目标对象类型校验
  - 如果set函数的第一个参数是 undefined/ null/ 原始类型值，那么会打印警告信息

S2 updateArrayEle: 更新 数组类型的成员
  - 进行参数校验：target是一个数组 + key是一个有效数组索引
  - target.splice(key, 1, val): 利用之前处理过的数组变异方法，更新/新增元素
  - 更新数组长度: 将数组长度修改为 target.length/key中的较大者，否则如果当要设置的元素的索引大于数组长度时, splice会无效

S3 updateHasDefiendPty: 更新 对象中已定义的属性
  - 已存在的属性经过处理，已经是响应式的了，所以直接更新值即可
  - 注意要 保证key在target上/在target的原型链上 + 必须不能在Object.pty上 [PS1]

S4 setNewValReactive: 创建 对象中未定义的属性值为响应式数据 + 触发ob依赖
  - 当target是 Vue实例对象/根数据对象时，是不允许添加属性的  [PS2]
  - 当target是非响应式时，简单赋值即可
  - 保证新添加的属性是响应式的 + 调用 __ob__.dep.notify 从而触发响应


[PS1] 不使用hasOwn(target, key)的原因
  - 其实就是in操作符和Object.prototype.hasOwnProperty的区别
  - 和in运算符不同，Object.pty.hasOwnProperty会忽略 从原型链上继承到的属性
  - 具体分析，见[issues](https://github.com/vuejs/vue/issues/6845) 

[PS2] target是根数据对象时，不允许添加属性的原因
```ts

  name: 'haha',
  info: {
    mid: 123,
  }
}

observe(data)
// data的伪代码为
data = {
  name: 'haha',
  // 记做ob1
  __ob__: {value: data, vmCount:1, dep: [] },

  info: {
    mid: 123,
    // 记做ob2
    __ob__: {value: info, vmCount:0, dep: [] }

/**
 * 从上文分析的流程可知:
 * ob.dep.depend()都是作为childOb.dep.depend()调用的；
 * 也就是说，代码中没有逻辑让 ob1.dep收集了依赖
 * 既然ob1.dep从未收集过依赖，也就不允许在data上新增属性了
 */
```

```ts
export function set (target: Array<any> | Object, key: any, val: any): any {
  //S1 目标对象类型校验
  if (process.env.NODE_ENV !== 'production' &&
  (isUndef(target) || isPrimitive(target))
  ) {
    warn(`Cannot set reactive property on undefined, null, or primitive value: ${(target)}`)
  }
  //S2 更新 数组类型的成员
  if (Array.isArray(target) && isValidArrayIndex(key)) {
    target.length = Math.max(target.length, key)
    target.splice(key, 1, val)
    return val
  }
  //S3 更新 对象中已定义的属性
  if (key in target && !(key in Object.prototype)) {
    target[key] = val
    return val
  }
   //S4 创建 对象中未定义的属性值为响应式数据 + 触发ob依赖
  const ob = (target: any).__ob__
  // 当target是 Vue实例对象/根数据对象时，是不允许添加属性的
  if (target._isVue || (ob && ob.vmCount)) {
    process.env.NODE_ENV !== 'production' && warn(
      'Avoid adding reactive properties to a Vue instance or its root $data ' +
      'at runtime - declare it upfront in the data option.'
    )
    return val
  }
  // 当target是非响应式时，简单赋值即可
  if (!ob) {
    target[key] = val
    return val
  }
  // 保证新添加的属性是响应式的 + 调用 __ob__.dep.notify 从而触发响应
  defineReactive(ob.value, key, val)
  ob.dep.notify()
  return val
}
```


### 3 Vue.delete/$delete

1 内部流程: <br/>
S1 checkTargetType: 目标对象类型校验
  - 如果set函数的第一个参数是 undefined/ null/ 原始类型值，那么会打印警告信息

S2 updateArrayEle: 更新 数组类型的成员
  - 进行参数校验：target是一个数组 + key是一个有效数组索引
  - target.splice(key, 1): 利用之前处理过的数组变异方法，删除元素

S3 readyDeleteVal: 准备删除属性
  - 当target是 Vue实例对象/根数据对象时，不允许删除属性
  - target不存在该属性则直接返回
  - 删除属性 + 触发ob依赖

---------------------------------------
2 代码实现:
```ts
export function del (target: Array<any> | Object, key: any) {
  //S1 目标对象类型校验
  if (process.env.NODE_ENV !== 'production' &&
    (isUndef(target) || isPrimitive(target))
  ) {
    warn(`Cannot delete reactive property on undefined, null, or primitive value: ${(target)}`)
  }
  //S2 更新 数组类型的成员
  if (Array.isArray(target) && isValidArrayIndex(key)) {
    target.splice(key, 1)
    return
  }
  //S3 准备删除属性
  const ob = (target: any).__ob__
  // 当target是 Vue实例对象/根数据对象时，不允许删除属性
  if (target._isVue || (ob && ob.vmCount)) {
    process.env.NODE_ENV !== 'production' && warn(
      'Avoid deleting properties on a Vue instance or its root $data ' +
      '- just set it to null.'
    )
    return
  }
  // target不存在该属性则直接返回
  if (!hasOwn(target, key)) {
    return
  }
  // 删除属性 + 触发ob依赖
  delete target[key]
  if (!ob) {
    return
  }
  ob.dep.notify()
}
```
