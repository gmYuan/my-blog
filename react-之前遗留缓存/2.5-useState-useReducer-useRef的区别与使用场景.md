# useState、useReducer、useRef的区别与使用场景

## 1. 层级关系图

```
React 组件需要"记忆"数据
         |
    ┌────┴────┐
    |         |
需要触发     不需要触发
组件更新     组件更新
    |         |
    ├─────┐   └──> useRef
    |     |
  数据简单  数据复杂/关联
    |      |
useState  useReducer
```

## 2. 各自的职责

### useState
- **职责**: 管理简单状态，状态变化触发组件重新渲染
- **输入**: 初始值
- **输出**: [当前值, 更新函数]
- **类比**: 电灯开关，按一下就变状态，房间立刻变亮/暗

### useReducer
- **职责**: 管理复杂状态或多个关联状态，通过 action 集中管理状态变更
- **输入**: reducer函数 + 初始值
- **输出**: [当前值, dispatch函数]
- **类比**: 遥控器，不同按钮触发不同动作，统一管理各种操作
- **注意**: 即使state值不变，也会触发重新渲染（与useState不同）

### useRef
- **职责**: 记忆数据但不触发渲染，常用于DOM引用或跨渲染保持值
- **输入**: 初始值
- **输出**: { current: 值 } 的可变对象
- **类比**: 笔记本，随时记录但不会影响房间的灯光

## 3. 调用链路

**场景：计数器 + 输入框 + DOM操作**

```
1. 用户点击按钮
   ↓
2. useState 更新 count → 触发重新渲染 → UI更新
   ↓
3. 用户输入文本（复杂表单）
   ↓
4. useReducer dispatch({type: 'UPDATE_FORM'}) → 触发重新渲染 → UI更新
   ↓
5. 记录点击次数（不影响UI）
   ↓
6. useRef.current++ → 不触发渲染 → UI不变
```

## 4. 为什么需要三种Hook？

**原因1**: 性能优化 - useRef 不触发渲染，避免不必要的性能开销

**原因2**: 代码可维护性 - useReducer 集中管理复杂状态逻辑
- 例子：购物车（商品数量、总价、优惠券）用useReducer比多个useState更清晰

## 5. 类比锚定

想象你在管理一个房间：
- **useState** 是墙上的开关，动一下全屋都知道（灯亮了）
- **useReducer** 是智能家居控制面板，按不同按钮执行不同场景（回家模式、睡眠模式），统一管理
- **useRef** 是你的口袋备忘录，随时记录但不会让房间发生变化

## 6. 代码示例

```jsx
import { useState, useReducer, useRef } from 'react';

function Example() {
  // useState: 简单计数
  const [count, setCount] = useState(0);
  
  // useReducer: 复杂表单
  const [form, dispatch] = useReducer(
    (state, action) => {
      switch(action.type) {
        case 'UPDATE_NAME': return {...state, name: action.value};
        case 'UPDATE_AGE': return {...state, age: action.value};
        case 'RESET': return {name: '', age: 0};
        default: return state;
      }
    },
    {name: '', age: 0}
  );
  
  // useRef: 记录点击次数（不触发渲染）
  const clickTimesRef = useRef(0);
  
  const handleClick = () => {
    setCount(count + 1);  // 触发渲染
    clickTimesRef.current++;  // 不触发渲染
    console.log('真实点击次数:', clickTimesRef.current);
  };
  
  return (
    <div>
      <p>显示的count: {count}</p>
      <button onClick={handleClick}>点击</button>
      
      <input 
        value={form.name}
        onChange={(e) => dispatch({type: 'UPDATE_NAME', value: e.target.value})}
      />
    </div>
  );
}
```

## 7. 深入理解

### useState vs useReducer：何时选择谁？

**判断标准**

| 场景 | 选择 | 原因 |
|------|------|------|
| 简单的布尔值、数字、字符串 | useState | 代码简洁 |
| 多个相关联的状态 | useReducer | 统一管理，避免不一致 |
| 状态更新逻辑复杂 | useReducer | 集中在reducer，易维护 |
| 需要状态历史记录 | useReducer | 可以记录action历史 |
| 状态需要传给深层组件 | useReducer | dispatch不变，避免回调地狱 |

**复杂表单的对比**

```jsx
// ❌ useState：代码分散，容易不一致
function Form() {
  const [name, setName] = useState('');
  const [age, setAge] = useState(0);
  const [email, setEmail] = useState('');
  const [errors, setErrors] = useState({});
  
  const handleNameChange = (e) => {
    setName(e.target.value);
    // 验证逻辑
    if (e.target.value.length < 2) {
      setErrors(prev => ({ ...prev, name: '名字太短' }));
    } else {
      setErrors(prev => {
        const { name, ...rest } = prev;
        return rest;
      });
    }
  };
  
  const handleReset = () => {
    setName('');
    setAge(0);
    setEmail('');
    setErrors({});
    // 容易忘记某个state
  };
}

// ✅ useReducer：逻辑集中，状态一致
function formReducer(state, action) {
  switch (action.type) {
    case 'UPDATE_FIELD':
      const newState = { ...state, [action.field]: action.value };
      // 统一验证逻辑
      return { ...newState, errors: validateForm(newState) };
    
    case 'RESET':
      return { name: '', age: 0, email: '', errors: {} };
    
    default:
      return state;
  }
}

function Form() {
  const [state, dispatch] = useReducer(formReducer, {
    name: '',
    age: 0,
    email: '',
    errors: {}
  });
  
  const handleChange = (field, value) => {
    dispatch({ type: 'UPDATE_FIELD', field, value });
  };
  
  const handleReset = () => {
    dispatch({ type: 'RESET' });
  };
}
```

### useState vs useReducer 的内部差异

**useState的state相等性检查**

```jsx
function Component() {
  const [obj, setObj] = useState({ count: 0 });
  
  const handleClick = () => {
    obj.count++;        // 直接修改
    setObj(obj);        // 传入相同引用
    // ✗ 组件不会重新渲染！React用Object.is比较，引用相同
  };
  
  const handleClickCorrect = () => {
    setObj({ ...obj, count: obj.count + 1 });  // 新对象
    // ✓ 组件会重新渲染
  };
}
```

**useReducer总是触发渲染**

```jsx
function Component() {
  const [state, dispatch] = useReducer(reducer, { count: 0 });
  
  const handleClick = () => {
    dispatch({ type: 'INCREMENT' });  // 即使state没变，也会渲染
  };
}

function reducer(state, action) {
  switch (action.type) {
    case 'INCREMENT':
      if (state.count >= 10) {
        return state;  // 返回相同引用
        // ✗ 但组件仍会重新渲染（与useState不同）
      }
      return { count: state.count + 1 };
  }
}
```

**源码差异**：
- useState内部调用`Object.is(oldState, newState)`，相等则跳过渲染
- useReducer会标记更新，即使state引用相同也会渲染

### useRef的三大使用场景

**场景1：访问DOM元素**

```jsx
function FocusInput() {
  const inputRef = useRef(null);
  
  useEffect(() => {
    // 自动聚焦
    inputRef.current.focus();
  }, []);
  
  return <input ref={inputRef} />;
}
```

**场景2：保存可变值（不触发渲染）**

```jsx
function Timer() {
  const [count, setCount] = useState(0);
  const timerRef = useRef(null);
  
  const start = () => {
    timerRef.current = setInterval(() => {
      setCount(c => c + 1);
    }, 1000);
  };
  
  const stop = () => {
    clearInterval(timerRef.current);
    // 如果用useState存储timerId，会触发不必要的渲染
  };
  
  return (
    <div>
      <p>{count}</p>
      <button onClick={start}>开始</button>
      <button onClick={stop}>停止</button>
    </div>
  );
}
```

**场景3：保存上一次的值**

```jsx
function usePrevious(value) {
  const ref = useRef();
  
  useEffect(() => {
    ref.current = value;  // 渲染后更新
  });
  
  return ref.current;  // 返回上一次的值
}

function Counter() {
  const [count, setCount] = useState(0);
  const prevCount = usePrevious(count);
  
  return (
    <div>
      <p>当前: {count}, 上一次: {prevCount}</p>
      <button onClick={() => setCount(count + 1)}>+1</button>
    </div>
  );
}
```

### useRef vs 闭包陷阱

**问题：闭包捕获旧值**

```jsx
function Component() {
  const [count, setCount] = useState(0);
  
  useEffect(() => {
    const timer = setInterval(() => {
      console.log(count);  // 永远打印0（闭包陷阱）
      setCount(count + 1); // 永远是0+1
    }, 1000);
    
    return () => clearInterval(timer);
  }, []);  // 依赖为空，useEffect只执行一次，闭包捕获初始count=0
}
```

**解决方案1：函数式更新**

```jsx
useEffect(() => {
  const timer = setInterval(() => {
    setCount(c => c + 1);  // c总是最新值
  }, 1000);
  
  return () => clearInterval(timer);
}, []);
```

**解决方案2：useRef存储最新值**

```jsx
function Component() {
  const [count, setCount] = useState(0);
  const countRef = useRef(count);
  
  useEffect(() => {
    countRef.current = count;  // 每次渲染更新ref
  }, [count]);
  
  useEffect(() => {
    const timer = setInterval(() => {
      console.log(countRef.current);  // 总是最新值
    }, 1000);
    
    return () => clearInterval(timer);
  }, []);
}
```

### useReducer的高级用法

**1. lazy初始化**

```jsx
function init(initialCount) {
  // 复杂计算，只执行一次
  return { count: initialCount, history: [] };
}

function reducer(state, action) {
  switch (action.type) {
    case 'INCREMENT':
      return {
        count: state.count + 1,
        history: [...state.history, state.count]
      };
    case 'RESET':
      return init(action.payload);
    default:
      return state;
  }
}

function Component({ initialCount }) {
  const [state, dispatch] = useReducer(reducer, initialCount, init);
  //                                            第三个参数是init函数
}
```

**2. 模拟Redux的中间件**

```jsx
function loggerMiddleware(dispatch) {
  return (action) => {
    console.log('dispatch:', action);
    const result = dispatch(action);
    console.log('new state:', result);
    return result;
  };
}

function Component() {
  const [state, originalDispatch] = useReducer(reducer, initialState);
  const dispatch = useMemo(
    () => loggerMiddleware(originalDispatch),
    [originalDispatch]
  );
}
```

**3. dispatch稳定性优化**

```jsx
// dispatch函数引用永不变，可以安全地传给子组件
function Parent() {
  const [state, dispatch] = useReducer(reducer, initialState);
  
  return (
    <Child 
      onUpdate={dispatch}  // dispatch引用永不变
      // 比传多个setState函数更好
    />
  );
}

const Child = memo(function Child({ onUpdate }) {
  // onUpdate不变，Child不会因父组件渲染而重新渲染
  return <button onClick={() => onUpdate({ type: 'UPDATE' })}>更新</button>;
});
```

### 性能对比

```jsx
// 场景：大型表单，多个字段
// 方案1：多个useState - 每次更新只影响一个state
const [name, setName] = useState('');
const [age, setAge] = useState(0);
const [email, setEmail] = useState('');
// 更新name时，age和email不会重新创建

// 方案2：单个useState对象 - 每次更新创建新对象
const [form, setForm] = useState({ name: '', age: 0, email: '' });
setForm({ ...form, name: 'new' });  // 整个对象浅拷贝

// 方案3：useReducer - 统一管理，dispatch不变
const [form, dispatch] = useReducer(reducer, initialState);
dispatch({ type: 'UPDATE_NAME', value: 'new' });  // dispatch引用稳定

// 结论：
// - 字段少且独立：多个useState
// - 字段多且关联：useReducer
// - 需要传递更新函数给子组件：useReducer（dispatch稳定）
```

## 8. 一句话总结

useState管理简单独立状态，useReducer集中管理复杂关联状态，useRef存储不触发渲染的可变值，三者配合覆盖所有状态管理场景。

## 9. 面试回答路径

### 问题：useState、useReducer、useRef有什么区别？

**回答结构（2分钟）**：

**第一步：概括三者定位**
> "这三个Hook都可以存储数据，但用途不同：useState管理简单状态会触发渲染，useReducer管理复杂状态会触发渲染，useRef存储数据不触发渲染。"

**第二步：详细说明useState**
> "useState适合简单的独立状态，比如布尔值、数字、字符串。特点是：state变化才重新渲染，通过Object.is判断是否变化。"

```jsx
const [count, setCount] = useState(0);
setCount(1);  // 触发渲染
setCount(1);  // 不触发渲染（值相同）
```

**第三步：详细说明useReducer**
> "useReducer适合复杂的关联状态，比如表单多个字段、状态有复杂更新逻辑。特点是：通过dispatch触发action，reducer集中处理逻辑。重要的是，即使返回相同引用也会渲染，这点和useState不同。"

```jsx
const [form, dispatch] = useReducer(reducer, { name: '', age: 0 });
dispatch({ type: 'UPDATE_NAME', value: '张三' });  // 逻辑在reducer里
```

**第四步：详细说明useRef**
> "useRef适合不需要触发渲染的数据，比如定时器ID、DOM引用、上一次的值。特点是：修改ref.current不会触发渲染，跨渲染保持引用。"

```jsx
const timerRef = useRef(null);
timerRef.current = setInterval(...);  // 不触发渲染
clearInterval(timerRef.current);      // 不触发渲染
```

**第五步：给出选择建议**
> "选择原则：需要触发渲染用useState或useReducer，不需要触发渲染用useRef。useState和useReducer之间，简单独立状态用useState，复杂关联状态或需要传dispatch给子组件用useReducer。"

### 问题：什么时候用useReducer替代useState？

**回答结构（2分钟）**：

**第一步：列出判断标准**
> "有四种情况建议用useReducer：状态逻辑复杂、多个状态相关联、需要传更新函数给深层组件、需要状态历史记录。"

**第二步：场景1 - 状态逻辑复杂**
> "比如表单验证，每次更新字段要检查多个条件、更新错误提示。用useState要写很多if-else，用useReducer可以集中在reducer里。"

```jsx
// useState：逻辑分散
const handleNameChange = (value) => {
  setName(value);
  if (value.length < 2) {
    setErrors({ ...errors, name: '太短' });
  } else if (value.length > 20) {
    setErrors({ ...errors, name: '太长' });
  } else {
    const { name, ...rest } = errors;
    setErrors(rest);
  }
};

// useReducer：逻辑集中
function reducer(state, action) {
  if (action.type === 'UPDATE_NAME') {
    const newState = { ...state, name: action.value };
    return { ...newState, errors: validateForm(newState) };
  }
}
```

**第三步：场景2 - 多个状态关联**
> "比如购物车，修改商品数量要同时更新总价、优惠金额、最终价格。用多个useState容易不一致，用useReducer保证一次更新所有关联状态。"

```jsx
function cartReducer(state, action) {
  switch (action.type) {
    case 'UPDATE_QUANTITY':
      const items = state.items.map(item =>
        item.id === action.id ? { ...item, quantity: action.quantity } : item
      );
      const subtotal = calculateSubtotal(items);
      const discount = calculateDiscount(subtotal);
      return {
        items,
        subtotal,
        discount,
        total: subtotal - discount
      };
  }
}
```

**第四步：场景3 - dispatch传递**
> "如果需要把更新函数传给很多子组件，useReducer的dispatch引用永不变，避免useCallback包裹多个setState。"

**第五步：给出经验法则**
> "经验法则：单个useState超过5行更新逻辑，或者有3个以上相关联的useState，就考虑合并成useReducer。"

### 问题：useRef有什么用？和useState有什么区别？

**回答结构（2分钟）**：

**第一步：说明核心区别**
> "核心区别是：修改useState触发重新渲染，修改useRef不触发渲染。useRef返回一个可变对象，修改current属性不会让组件重新渲染。"

**第二步：列出三大使用场景**

**场景1：访问DOM**
```jsx
const inputRef = useRef(null);
inputRef.current.focus();  // 操作DOM
```

**场景2：存储定时器ID等不需要渲染的值**
```jsx
const timerRef = useRef(null);
timerRef.current = setInterval(...);  // 存储ID
clearInterval(timerRef.current);      // 清理
// 如果用useState存储ID，每次setTimerId都会触发不必要的渲染
```

**场景3：保存上一次的值**
```jsx
function usePrevious(value) {
  const ref = useRef();
  useEffect(() => {
    ref.current = value;  // 渲染后更新
  });
  return ref.current;  // 返回上一次
}
```

**第三步：解释闭包陷阱**
> "useRef还能解决闭包陷阱问题。定时器、事件监听器中访问state，拿到的是旧值（闭包捕获）。用useRef存储最新值，ref.current总是最新的。"

```jsx
// 问题：闭包陷阱
useEffect(() => {
  setInterval(() => {
    console.log(count);  // 永远是0
  }, 1000);
}, []);

// 解决：useRef
const countRef = useRef(count);
useEffect(() => { countRef.current = count; }, [count]);
useEffect(() => {
  setInterval(() => {
    console.log(countRef.current);  // 总是最新值
  }, 1000);
}, []);
```

**第四步：选择建议**
> "选择原则：变化需要反映到UI用useState，不需要UI响应用useRef。比如记录滚动位置、存储定时器ID、缓存计算结果等都用useRef。"

### 问题：useState的state不变时不渲染，useReducer为什么总是渲染？

**回答结构（90秒）**：

**第一步：说明现象**
> "useState用Object.is比较新旧state，相等就跳过渲染。useReducer即使返回相同引用也会触发渲染。"

```jsx
// useState：相同引用不渲染
const [obj, setObj] = useState({ count: 0 });
setObj(obj);  // 不渲染

// useReducer：相同引用也渲染
const [state, dispatch] = useReducer(reducer, { count: 0 });
function reducer(state, action) {
  return state;  // 返回相同引用
}
dispatch({ type: 'ANY' });  // 仍然渲染
```

**第二步：解释原因**
> "这是React的设计选择。useState假设你会返回新对象，所以做相等性检查优化。useReducer假设每次dispatch都是有意义的更新，所以不做检查，直接标记更新。"

**第三步：说明影响**
> "实际影响不大，因为正确使用时，reducer应该返回新对象。如果确实想优化，可以在reducer里手动检查是否需要更新。"

```jsx
function reducer(state, action) {
  switch (action.type) {
    case 'INCREMENT':
      if (state.count >= 10) {
        return state;  // 返回旧state，但仍会渲染
      }
      return { ...state, count: state.count + 1 };
  }
}

// 优化：外层用React.memo
const Component = memo(function({ value }) {
  const [state, dispatch] = useReducer(reducer, initialState);
  // memo会阻止props相同时的渲染
});
```

**第四步：最佳实践**
> "最佳实践是：reducer总是返回新对象（即使值相同），保持immutable更新。如果有性能问题，用React.memo包裹组件，而不是在reducer里返回旧state。"
