# useMemo与useCallback的作用与使用场景

## 1. 层级关系图

```
性能优化需求
     |
  需要缓存？
     |
  ┌──┴──┐
  |     |
缓存值  缓存函数
  |     |
useMemo useCallback
  |     |
避免重复  避免子组件
计算耗时  不必要渲染
```

## 2. 各自的职责

### useMemo
- **职责**: 缓存计算结果，依赖不变时返回缓存值
- **输入**: 计算函数 + 依赖数组
- **输出**: 缓存的计算结果
- **类比**: 计算器的记忆功能，相同输入直接返回结果，不重新计算

### useCallback
- **职责**: 缓存函数引用，依赖不变时返回同一个函数
- **输入**: 函数 + 依赖数组
- **输出**: 缓存的函数引用
- **类比**: 电话快捷键，只要对方号码没变，按快捷键拨的就是同一个号码

## 3. 调用链路

**场景：列表筛选 + 子组件传递函数**

```
1. 父组件渲染，有大量数据需要筛选
   ↓
2. useMemo 检查依赖是否变化
   ↓
3. 依赖未变 → 返回缓存结果，跳过计算
   依赖改变 → 重新执行计算，缓存新结果
   ↓
4. 父组件传递函数给子组件
   ↓
5. useCallback 检查依赖是否变化
   ↓
6. 依赖未变 → 返回同一函数引用 → 子组件props未变 → 子组件不渲染
   依赖改变 → 返回新函数 → 子组件重新渲染
```

## 4. 为什么需要这两个Hook？

**原因1**: 避免重复计算 - 复杂计算很耗时，缓存结果提升性能
- 例子：10万条数据筛选，如果每次渲染都重新筛选，会卡顿

**原因2**: 避免子组件无效渲染 - React.memo的子组件依赖props引用相等判断
- 例子：父组件传函数给子组件，不用useCallback每次都是新函数，子组件就会重新渲染

## 5. 类比锚定

想象你在餐厅点餐：
- **useMemo** 是厨师的"今日特价"菜，材料相同就直接端出昨天做好的，不用重新炒（计算）
- **useCallback** 是服务员的"老客户识别"，只要是老客户就用同一张会员卡，新客户才发新卡（新函数引用）

## 6. 代码示例

```jsx
import { useMemo, useCallback, memo } from 'react';

// 子组件用 memo 包裹，props不变就不渲染
const ChildButton = memo(({ onClick, label }) => {
  console.log('子组件渲染了');
  return <button onClick={onClick}>{label}</button>;
});

function Parent() {
  const [count, setCount] = useState(0);
  const [filter, setFilter] = useState('');
  const [list] = useState(Array.from({length: 10000}, (_, i) => i));
  
  // useMemo: 缓存筛选结果，filter不变就不重新计算
  const filteredList = useMemo(() => {
    console.log('执行筛选计算');
    return list.filter(item => item.toString().includes(filter));
  }, [filter, list]);
  
  // useCallback: 缓存函数引用，count不变子组件就不会重新渲染
  const handleChildClick = useCallback(() => {
    console.log('点击了', count);
  }, [count]);
  
  return (
    <div>
      <input value={filter} onChange={(e) => setFilter(e.target.value)} />
      <p>筛选结果: {filteredList.length}条</p>
      
      <button onClick={() => setCount(count + 1)}>父组件count: {count}</button>
      
      {/* 如果不用useCallback，每次父组件渲染，子组件都会重新渲染 */}
      <ChildButton onClick={handleChildClick} label="子组件按钮" />
    </div>
  );
}
```

## 7. 深入理解

### 过度优化的陷阱

**并非所有函数都需要useCallback**

```jsx
// ❌ 过度优化：浪费资源
function Component() {
  const [count, setCount] = useState(0);
  
  // 这个函数只在当前组件用，不传给子组件，不需要useCallback
  const handleClick = useCallback(() => {
    setCount(count + 1);
  }, [count]);
  
  // useCallback本身有成本：
  // 1. 创建闭包
  // 2. 依赖比较
  // 3. 内存占用
  // 如果没有明显收益，反而浪费
  
  return <button onClick={handleClick}>{count}</button>;
}

// ✅ 简单直接：创建新函数的成本很低
function Component() {
  const [count, setCount] = useState(0);
  
  const handleClick = () => {
    setCount(count + 1);
  };
  
  return <button onClick={handleClick}>{count}</button>;
}
```

**何时使用useCallback？**

1. **传递给用memo包裹的子组件**
```jsx
const Child = memo(function Child({ onClick }) {
  console.log('Child渲染');
  return <button onClick={onClick}>点击</button>;
});

function Parent() {
  const [count, setCount] = useState(0);
  
  // 必须用useCallback，否则Child每次都渲染
  const handleClick = useCallback(() => {
    console.log('clicked');
  }, []);
  
  return (
    <>
      <button onClick={() => setCount(count + 1)}>Count: {count}</button>
      <Child onClick={handleClick} />
    </>
  );
}
```

2. **作为useEffect的依赖**
```jsx
function Component({ id }) {
  // 如果fetchData每次都是新函数，useEffect会重复执行
  const fetchData = useCallback(async () => {
    const data = await fetch(`/api/${id}`);
    return data;
  }, [id]);
  
  useEffect(() => {
    fetchData();
  }, [fetchData]);  // fetchData作为依赖
}
```

3. **作为自定义Hook的返回值**
```jsx
function useToggle(initial) {
  const [value, setValue] = useState(initial);
  
  // 返回的函数应该稳定，避免使用者的useEffect重复执行
  const toggle = useCallback(() => {
    setValue(v => !v);
  }, []);
  
  return [value, toggle];
}
```

### useMemo vs 计算属性

**React没有Vue的计算属性，用useMemo模拟**

```jsx
// Vue的计算属性
computed: {
  filteredList() {
    return this.list.filter(item => item.includes(this.filter));
  }
}

// React用useMemo
function Component() {
  const [list, setList] = useState([]);
  const [filter, setFilter] = useState('');
  
  const filteredList = useMemo(() => {
    return list.filter(item => item.includes(filter));
  }, [list, filter]);
}

// 不用useMemo的问题
function Component() {
  const [list, setList] = useState([]);
  const [filter, setFilter] = useState('');
  const [count, setCount] = useState(0);
  
  // 每次渲染都重新计算，即使list和filter没变
  const filteredList = list.filter(item => item.includes(filter));
  
  return (
    <>
      <button onClick={() => setCount(count + 1)}>Count: {count}</button>
      {/* count变化导致filteredList重新计算，浪费 */}
    </>
  );
}
```

### 依赖数组的常见陷阱

**陷阱1：遗漏依赖**

```jsx
function Component({ userId }) {
  const [user, setUser] = useState(null);
  
  // ❌ 遗漏userId依赖
  const fetchUser = useCallback(async () => {
    const data = await fetch(`/api/users/${userId}`);
    setUser(data);
  }, []);  // ESLint警告：遗漏userId
  
  // userId变化时，fetchUser仍使用旧的userId（闭包陷阱）
}

// ✅ 添加完整依赖
const fetchUser = useCallback(async () => {
  const data = await fetch(`/api/users/${userId}`);
  setUser(data);
}, [userId]);
```

**陷阱2：依赖对象或数组**

```jsx
function Component({ config }) {
  // ❌ config是对象，每次父组件渲染都是新对象
  const result = useMemo(() => {
    return expensiveCalculation(config);
  }, [config]);  // config引用变化，缓存失效
}

// ✅ 解构依赖的属性
function Component({ config }) {
  const result = useMemo(() => {
    return expensiveCalculation(config);
  }, [config.a, config.b, config.c]);  // 只依赖具体属性
}

// ✅ 父组件用useMemo稳定config
function Parent() {
  const config = useMemo(() => ({ a: 1, b: 2 }), []);
  return <Child config={config} />;
}
```

**陷阱3：函数依赖**

```jsx
function Component({ onSuccess }) {
  // ❌ onSuccess每次都是新函数
  const handleClick = useCallback(() => {
    doSomething();
    onSuccess();
  }, [onSuccess]);  // onSuccess变化，handleClick失效
}

// ✅ 父组件用useCallback包裹onSuccess
function Parent() {
  const handleSuccess = useCallback(() => {
    console.log('success');
  }, []);
  
  return <Child onSuccess={handleSuccess} />;
}
```

### 性能测量：何时真正需要优化？

**React DevTools Profiler测量**

```jsx
// 先写简单版本
function Component() {
  const [count, setCount] = useState(0);
  const [list, setList] = useState(hugeArray);
  
  const filtered = list.filter(item => item.value > 100);
  
  return <div>{filtered.length}</div>;
}

// 用Profiler测量渲染时间
// 如果filtered计算很快(<1ms)，不需要优化
// 如果很慢(>10ms)，加useMemo

const filtered = useMemo(() => {
  return list.filter(item => item.value > 100);
}, [list]);
```

**经验法则**：
- 计算复杂度O(n)且n<1000：不需要useMemo
- 计算复杂度O(n²)或n>10000：考虑useMemo
- 函数传给5个以内的子组件：不需要useCallback
- 函数传给很多子组件或作为Context value：需要useCallback

### useMemo的高级用法

**1. 缓存组件**

```jsx
function Parent() {
  const [count, setCount] = useState(0);
  const [text, setText] = useState('');
  
  // 缓存整个子组件，text不变就不重新渲染
  const child = useMemo(() => {
    return <ExpensiveChild text={text} />;
  }, [text]);
  
  return (
    <div>
      <button onClick={() => setCount(count + 1)}>Count: {count}</button>
      {child}
    </div>
  );
}
```

**2. 缓存Context value**

```jsx
function Provider({ children }) {
  const [user, setUser] = useState(null);
  const [theme, setTheme] = useState('light');
  
  // 缓存value对象，避免Consumer不必要渲染
  const value = useMemo(() => ({
    user,
    theme,
    setUser,
    setTheme
  }), [user, theme]);
  
  return (
    <MyContext.Provider value={value}>
      {children}
    </MyContext.Provider>
  );
}
```

**3. 缓存复杂计算**

```jsx
function Component({ data }) {
  // 斐波那契数列（递归计算慢）
  const fibonacci = useMemo(() => {
    const fib = (n) => n <= 1 ? n : fib(n - 1) + fib(n - 2);
    return fib(data.n);
  }, [data.n]);
  
  // 数组排序（大数组慢）
  const sorted = useMemo(() => {
    return [...data.list].sort((a, b) => b.score - a.score);
  }, [data.list]);
  
  return <div>{fibonacci} - {sorted[0]?.name}</div>;
}
```

### useCallback vs useMemo 的关系

```jsx
// useCallback是useMemo的语法糖
const handleClick = useCallback(() => {
  console.log('clicked');
}, []);

// 等价于
const handleClick = useMemo(() => {
  return () => {
    console.log('clicked');
  };
}, []);

// 所以useCallback(() => fn, deps)
// 等价于 useMemo(() => fn, deps)
```

## 8. 一句话总结

useMemo缓存计算结果避免重复计算，useCallback缓存函数引用避免子组件无效渲染，需要配合React.memo使用，避免过度优化。

## 9. 面试回答路径

### 问题：useMemo和useCallback有什么用？

**回答结构（2分钟）**：

**第一步：概括用途**
> "两者都是性能优化Hook。useMemo缓存计算结果，useCallback缓存函数引用，避免不必要的重复计算和组件渲染。"

**第二步：详细说明useMemo**
> "useMemo用于缓存计算结果。当组件重新渲染时，如果依赖没变，直接返回缓存的值，不重新计算。"

```jsx
const filteredList = useMemo(() => {
  console.log('计算中');
  return list.filter(item => item.includes(filter));
}, [list, filter]);

// filter不变时，不打印'计算中'，直接返回缓存值
```

> "适用场景：大数组过滤排序、复杂计算、递归算法等耗时操作。"

**第三步：详细说明useCallback**
> "useCallback用于缓存函数引用。每次渲染都会创建新函数，如果传给子组件，会导致子组件不必要渲染。"

```jsx
// 不用useCallback，每次都是新函数
const handleClick = () => { console.log('clicked'); };

// 用useCallback，依赖不变就返回相同函数
const handleClick = useCallback(() => {
  console.log('clicked');
}, []);

// 配合React.memo使用
const Child = memo(({ onClick }) => <button onClick={onClick} />);
```

**第四步：说明使用条件**
> "重要的是，useCallback要配合React.memo才有意义。如果子组件没用memo包裹，父组件的props变化都会触发子组件渲染，useCallback就没用了。"

**第五步：避免过度优化**
> "不是所有函数都要用useCallback。如果函数只在当前组件用，不传给子组件，不需要优化。useCallback本身也有成本，过度使用反而降低性能。"

### 问题：为什么useCallback要配合React.memo使用？

**回答结构（90秒）**：

**第一步：说明React的默认行为**
> "React默认情况下，父组件渲染会导致所有子组件重新渲染，无论props是否变化。React.memo可以阻止props相同时的渲染。"

```jsx
// 没有memo的子组件
function Child({ onClick }) {
  console.log('Child渲染');
  return <button onClick={onClick}>点击</button>;
}

function Parent() {
  const [count, setCount] = useState(0);
  const handleClick = () => console.log('clicked');
  
  return (
    <>
      <button onClick={() => setCount(count + 1)}>Count: {count}</button>
      <Child onClick={handleClick} />
      {/* count变化，Parent渲染，Child也渲染（即使props没变） */}
    </>
  );
}
```

**第二步：说明memo的作用**
> "React.memo包裹后，会浅比较props，相同就跳过渲染。但函数每次渲染都是新的，引用不同，memo认为props变了。"

```jsx
const Child = memo(function Child({ onClick }) {
  console.log('Child渲染');
  return <button onClick={onClick}>点击</button>;
});

function Parent() {
  const [count, setCount] = useState(0);
  const handleClick = () => console.log('clicked');  // 每次新函数
  
  return (
    <>
      <button onClick={() => setCount(count + 1)}>Count: {count}</button>
      <Child onClick={handleClick} />
      {/* 虽然有memo，但handleClick每次都是新函数，Child仍然渲染 */}
    </>
  );
}
```

**第三步：说明useCallback的作用**
> "useCallback保证函数引用稳定，配合memo才能真正阻止渲染。"

```jsx
function Parent() {
  const [count, setCount] = useState(0);
  
  const handleClick = useCallback(() => {
    console.log('clicked');
  }, []);  // 函数引用稳定
  
  return (
    <>
      <button onClick={() => setCount(count + 1)}>Count: {count}</button>
      <Child onClick={handleClick} />
      {/* count变化，handleClick引用不变，memo生效，Child不渲染 */}
    </>
  );
}
```

**第四步：总结**
> "所以useCallback和memo是配套的：memo负责比较props，useCallback负责稳定函数引用。缺一不可。"

### 问题：useMemo的依赖数组为空会怎样？

**回答结构（90秒）**：

**第一步：说明行为**
> "依赖数组为空`[]`时，useMemo的计算函数只在首次渲染时执行一次，之后永远返回缓存值。"

```jsx
const value = useMemo(() => {
  console.log('计算中');
  return expensiveCalculation();
}, []);  // 只打印一次'计算中'
```

**第二步：适用场景**
> "适合初始化时计算一次、之后不变的值。比如复杂的初始数据处理、固定的配置对象等。"

```jsx
// 场景1：处理props初始值
function Component({ initialData }) {
  const processedData = useMemo(() => {
    return initialData.map(item => ({
      ...item,
      computed: heavyComputation(item)
    }));
  }, []);  // 只处理一次，后续initialData变化不重新处理
}

// 场景2：稳定的对象引用
function Component() {
  const config = useMemo(() => ({
    timeout: 5000,
    retries: 3,
    headers: { 'Content-Type': 'application/json' }
  }), []);  // 对象引用永不变
  
  return <Child config={config} />;
}
```

**第三步：注意事项**
> "要注意，如果计算依赖外部变量，但依赖数组为空，会形成闭包陷阱，永远使用初始值。"

```jsx
function Component({ userId }) {
  // ❌ 错误：userId变化，但fetchUrl不变
  const fetchUrl = useMemo(() => {
    return `/api/users/${userId}`;
  }, []);  // userId=1时计算，之后永远是/api/users/1
  
  // ✅ 正确：添加userId依赖
  const fetchUrl = useMemo(() => {
    return `/api/users/${userId}`;
  }, [userId]);  // userId变化时重新计算
}
```

**第四步：对比useState的lazy初始化**
> "空依赖的useMemo类似useState的lazy初始化，都是只计算一次。但useState用于状态，useMemo用于派生值。"

```jsx
// useState lazy初始化
const [value] = useState(() => expensiveInit());  // 只执行一次

// useMemo空依赖
const value = useMemo(() => expensiveInit(), []);  // 只执行一次

// 区别：useState返回[value, setValue]，useMemo只返回value
```

### 问题：如何判断是否需要使用useMemo/useCallback？

**回答结构（2分钟）**：

**第一步：给出判断原则**
> "遵循'先写简单代码，遇到性能问题再优化'原则。不要一开始就到处用useMemo/useCallback，过度优化反而降低可读性和性能。"

**第二步：useMemo的使用时机**

**需要用：**
1. **计算复杂度高**：大数组排序、递归计算、复杂正则等
2. **数据量大**：处理10000+条数据
3. **计算频繁**：组件频繁渲染且每次都重新计算

```jsx
// 需要：大数组过滤
const filtered = useMemo(() => {
  return hugeArray.filter(item => item.score > 90);
}, [hugeArray]);

// 不需要：简单计算
const total = price * quantity;  // 不要用useMemo包这个
```

**第三步：useCallback的使用时机**

**需要用：**
1. **传给memo包裹的子组件**
2. **作为useEffect的依赖**
3. **作为自定义Hook的返回值**
4. **作为Context的value（如果包含函数）**

```jsx
// 需要：传给memo子组件
const Child = memo(({ onClick }) => <button onClick={onClick} />);
const handleClick = useCallback(() => {...}, []);
<Child onClick={handleClick} />

// 不需要：只在当前组件用
const handleClick = () => {...};  // 直接定义即可
<button onClick={handleClick} />
```

**第四步：性能测量方法**
> "使用React DevTools的Profiler测量渲染时间。如果渲染时间<16ms（60fps），不需要优化。如果>16ms且影响用户体验，再优化。"

**第五步：经验法则**
> "经验法则：先不用，等遇到实际性能问题再加。代码评审时如果看到无意义的useMemo/useCallback，应该移除。保持代码简单优先，性能优化其次。"
