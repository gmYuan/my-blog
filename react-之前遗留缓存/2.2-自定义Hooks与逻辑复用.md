# 自定义Hooks与逻辑复用

## 1. 层级关系图

```
逻辑复用方案演变
        |
    ┌───┴───┐
  过去      现在
    |       |
  mixin   自定义Hooks
  HOC        |
    |     简单直观
  复杂     显式调用
  冲突     无副作用
```

## 2. 各自的职责

### 自定义Hooks
- **职责**: 抽取可复用的状态逻辑，以use开头的函数
- **输入**: 可能需要参数
- **输出**: 状态、函数、或任意值
- **类比**: 工具箱，拿出来就能用，清楚来源

### mixin（已废弃）
- **职责**: 将多个对象的方法/状态合并到组件
- **问题**: 命名冲突、依赖不透明、来源不清
- **类比**: 杂物堆，东西混在一起，不知道哪个是哪个

### HOC（高阶组件）
- **职责**: 接收组件返回新组件，注入props
- **问题**: 嵌套地狱、props来源不明、调试困难
- **类比**: 套娃，一层包一层，打开才知道里面是什么

## 3. 调用链路

**场景：多个组件需要获取鼠标位置**

### 自定义Hooks方式
```
1. 定义 useMousePosition Hook
   ↓
2. Hook内部用 useState 存储位置
   ↓
3. Hook内部用 useEffect 监听 mousemove
   ↓
4. 组件调用 const position = useMousePosition()
   ↓
5. 直接使用 position.x 和 position.y
   ↓
6. 多个组件调用，各自独立状态，不冲突
```

### HOC方式（对比）
```
1. 定义 withMousePosition(Component)
   ↓
2. 返回包裹后的新组件
   ↓
3. 新组件内部监听 mousemove，通过props传给原组件
   ↓
4. 使用: export default withAuth(withTheme(withMouse(MyComponent)))
   ↓
5. 嵌套多了后，props来源不清，调试困难
```

## 4. 为什么用自定义Hooks？

**原因1**: 解决mixin的问题 - 显式调用，来源清晰，无命名冲突
- 例子：`const [a, setA] = useMyHook()` 一眼看出数据来源，变量名自己定不冲突

**原因2**: 比HOC更简洁 - 无需嵌套组件，调试方便
- 例子：需要3个能力时，Hooks是3行代码，HOC是3层嵌套

## 5. 类比锚定

想象你需要工具修东西：
- **mixin** 是把所有工具扔一个箱子，拿出来不知道是谁的，容易搞混
- **HOC** 是每个工具都套一个盒子，拆3层才能用，麻烦
- **自定义Hooks** 是工具分类摆好，贴上标签，拿来即用，一目了然

## 6. 代码示例

### 自定义Hook - 鼠标位置

```jsx
import { useState, useEffect } from 'react';

// 定义自定义Hook
function useMousePosition() {
  const [position, setPosition] = useState({ x: 0, y: 0 });
  
  useEffect(() => {
    const handleMove = (e) => {
      setPosition({ x: e.clientX, y: e.clientY });
    };
    
    window.addEventListener('mousemove', handleMove);
    return () => window.removeEventListener('mousemove', handleMove);
  }, []);
  
  return position;  // 返回状态
}

// 使用：简洁明了
function ComponentA() {
  const position = useMousePosition();
  return <div>鼠标位置: {position.x}, {position.y}</div>;
}

function ComponentB() {
  const { x, y } = useMousePosition();
  return <div>X: {x}, Y: {y}</div>;
}
```

### 自定义Hook - 本地存储

```jsx
function useLocalStorage(key, initialValue) {
  const [value, setValue] = useState(() => {
    const saved = localStorage.getItem(key);
    return saved ? JSON.parse(saved) : initialValue;
  });
  
  useEffect(() => {
    localStorage.setItem(key, JSON.stringify(value));
  }, [key, value]);
  
  return [value, setValue];
}

// 使用
function Settings() {
  const [theme, setTheme] = useLocalStorage('theme', 'light');
  const [lang, setLang] = useLocalStorage('lang', 'zh');
  
  return (
    <div>
      <button onClick={() => setTheme(theme === 'light' ? 'dark' : 'light')}>
        切换主题: {theme}
      </button>
      <button onClick={() => setLang(lang === 'zh' ? 'en' : 'zh')}>
        切换语言: {lang}
      </button>
    </div>
  );
}
```

```jsx
// 使用 - 嵌套地狱

// props来源不清晰

  // mousePosition从哪来？要看HOC代码才知道

```

### mixin为什么被废弃？

```js
// mixin写法（已废弃，仅作对比）
const MouseMixin = {
  componentDidMount() {
    // 如果多个mixin都有componentDidMount，容易冲突
  },
  getMousePosition() {
    // 方法名容易和其他mixin冲突
  }
};

// 使用
const MyComponent = React.createClass({
  mixins: [MouseMixin, AnotherMixin],  // 多个mixin，不知道哪个方法来自哪个
  // ...
});
```

**mixin问题总结**：
1. 依赖`React.createClass`（已移除）
2. 多个mixin的方法/state可能同名冲突
3. mixin之间可能有隐式依赖，调用顺序不清晰
4. 组件里用了某个方法/state，不知道来自哪个mixin

## 7. 深入理解

### use前缀的严格要求

**为什么必须use开头？**

```jsx
// ❌ 错误：不以use开头
function getWindowSize() {
  const [size, setSize] = useState({ width: 0, height: 0 });
  // ESLint报错：React Hook "useState" is called in function "getWindowSize" 
  // that is neither a React function component nor a custom React Hook function
}

// ✅ 正确：use开头
function useWindowSize() {
  const [size, setSize] = useState({ width: 0, height: 0 });
  // ESLint通过检查
}
```

**原因**：
1. **ESLint规则识别**：`eslint-plugin-react-hooks`通过函数名判断是否是Hook，检查Hooks规则
2. **团队约定**：一眼看出哪些函数包含状态逻辑
3. **工具支持**：IDE可以提供更好的自动完成和类型推导

### 自定义Hooks vs 普通函数

| 对比项 | 自定义Hooks | 普通函数 |
|-------|------------|---------|
| **命名** | 必须use开头 | 任意 |
| **可以调用Hooks** | ✓ 可以 | ✗ 不可以 |
| **返回值** | 状态、函数等 | 任意 |
| **执行时机** | 每次组件渲染 | 调用时执行 |
| **状态隔离** | 每次调用独立状态 | 无状态 |

```jsx
// 普通函数：纯计算，无副作用
function formatPrice(price) {
  return `¥${price.toFixed(2)}`;
}

// 自定义Hook：包含状态和副作用
function usePrice(initialPrice) {
  const [price, setPrice] = useState(initialPrice);
  
  useEffect(() => {
    // 副作用：订阅价格变化
    const unsubscribe = PriceAPI.subscribe(setPrice);
    return unsubscribe;
  }, []);
  
  return [price, setPrice];
}
```

### 自定义Hooks的最佳实践

**1. 状态隔离：每次调用独立状态**

```jsx
function useCounter() {
  const [count, setCount] = useState(0);
  const increment = () => setCount(c => c + 1);
  return { count, increment };
}

function App() {
  const counter1 = useCounter();  // 独立状态
  const counter2 = useCounter();  // 独立状态
  
  return (
    <>
      <div>Counter1: {counter1.count} <button onClick={counter1.increment}>+</button></div>
      <div>Counter2: {counter2.count} <button onClick={counter2.increment}>+</button></div>
      {/* counter1和counter2互不影响 */}
    </>
  );
}
```

**2. 返回对象 vs 返回数组**

```jsx
// 返回数组：适合少量返回值，可自定义命名
function useToggle(initial) {
  const [value, setValue] = useState(initial);
  const toggle = () => setValue(v => !v);
  return [value, toggle];  // 可以重命名：const [isOpen, setIsOpen]
}

// 返回对象：适合多个返回值，命名固定
function useUser(userId) {
  const [user, setUser] = useState(null);
  const [loading, setLoading] = useState(false);
  const [error, setError] = useState(null);
  
  return { user, loading, error, setUser };  // 必须用固定名字
}
```

**3. 依赖管理**

```jsx
// ❌ 错误：遗漏依赖
function useDebounce(value, delay) {
  const [debouncedValue, setDebouncedValue] = useState(value);
  
  useEffect(() => {
    const timer = setTimeout(() => {
      setDebouncedValue(value);
    }, delay);
    
    return () => clearTimeout(timer);
  }, [value]);  // ✗ 遗漏delay依赖
}

// ✅ 正确：完整依赖
function useDebounce(value, delay) {
  const [debouncedValue, setDebouncedValue] = useState(value);
  
  useEffect(() => {
    const timer = setTimeout(() => {
      setDebouncedValue(value);
    }, delay);
    
    return () => clearTimeout(timer);
  }, [value, delay]);  // ✓ 包含所有依赖
}
```

### 复杂自定义Hooks示例

**useAsync - 处理异步请求**

```jsx
function useAsync(asyncFunction, immediate = true) {
  const [status, setStatus] = useState('idle');
  const [data, setData] = useState(null);
  const [error, setError] = useState(null);
  
  const execute = useCallback(async (...params) => {
    setStatus('pending');
    setData(null);
    setError(null);
    
    try {
      const response = await asyncFunction(...params);
      setData(response);
      setStatus('success');
      return response;
    } catch (error) {
      setError(error);
      setStatus('error');
      throw error;
    }
  }, [asyncFunction]);
  
  useEffect(() => {
    if (immediate) {
      execute();
    }
  }, [execute, immediate]);
  
  return { execute, status, data, error, loading: status === 'pending' };
}

// 使用
function UserProfile({ userId }) {
  const { data: user, loading, error } = useAsync(
    () => fetchUser(userId),
    true
  );
  
  if (loading) return <div>Loading...</div>;
  if (error) return <div>Error: {error.message}</div>;
  return <div>{user.name}</div>;
}
```

**useLocalStorage - 同步本地存储**

```jsx
function useLocalStorage(key, initialValue) {
  // 延迟初始化：只在首次渲染读localStorage
  const [storedValue, setStoredValue] = useState(() => {
    try {
      const item = window.localStorage.getItem(key);
      return item ? JSON.parse(item) : initialValue;
    } catch (error) {
      console.error(error);
      return initialValue;
    }
  });
  
  const setValue = useCallback((value) => {
    try {
      const valueToStore = value instanceof Function ? value(storedValue) : value;
      setStoredValue(valueToStore);
      window.localStorage.setItem(key, JSON.stringify(valueToStore));
    } catch (error) {
      console.error(error);
    }
  }, [key, storedValue]);
  
  // 监听其他标签页的变化
  useEffect(() => {
    const handleStorage = (e) => {
      if (e.key === key && e.newValue) {
        setStoredValue(JSON.parse(e.newValue));
      }
    };
    
    window.addEventListener('storage', handleStorage);
    return () => window.removeEventListener('storage', handleStorage);
  }, [key]);
  
  return [storedValue, setValue];
}
```

### HOC的适用场景

**HOC不是完全被Hooks替代，某些场景仍然有用**

```jsx
// 场景1：增强props（类组件无法用Hooks）
function withLogger(Component) {
  return class extends React.Component {
    componentDidUpdate(prevProps) {
      console.log('Props changed:', prevProps, this.props);
    }
    
    render() {
      return <Component {...this.props} />;
    }
  };
}

// 场景2：条件渲染（权限控制）
function withAuth(Component) {
  return function AuthComponent(props) {
    const { user } = useAuth();
    
    if (!user) {
      return <Navigate to="/login" />;
    }
    
    return <Component {...props} />;
  };
}

// Hooks实现：更灵活
function ProtectedRoute() {
  const { user } = useAuth();
  
  if (!user) {
    return <Navigate to="/login" />;
  }
  
  return <Dashboard />;
}
```

## 8. 一句话总结

自定义Hooks用use开头抽取可复用的状态逻辑，通过闭包和调用约定实现状态隔离，比HOC和mixin更显式、可组合。

## 9. 面试回答路径

### 问题：什么是自定义Hooks？有什么用？

**回答结构（2分钟）**：

**第一步：定义**
> "自定义Hooks是以use开头的函数，内部可以调用其他Hooks，用于抽取和复用组件的状态逻辑。"

**第二步：解决的问题**
> "解决了React中逻辑复用的问题。以前用HOC或render props复用逻辑，会导致组件嵌套很深，代码难读。自定义Hooks可以扁平化地复用逻辑。"

**第三步：举例说明**

```jsx
// 需求：多个组件需要窗口宽度
// HOC方式：嵌套
const MyComponent = withWindowSize(withAuth(Component));

// Hooks方式：扁平
function MyComponent() {
  const windowSize = useWindowSize();
  const user = useAuth();
  // 一目了然
}
```

**第四步：说明优势**
> "自定义Hooks有三个优势：一是调用关系显式，`const data = useMyHook()`一眼看出数据来源；二是没有命名冲突，每次调用返回独立状态；三是可以组合，一个Hook内部可以调用其他Hooks。"

**第五步：补充规则**
> "使用时要遵守Hooks规则：函数名必须use开头，只能在函数组件或自定义Hook顶层调用，不能在循环、条件、嵌套函数中调用。"

### 问题：自定义Hooks和普通函数有什么区别？

**回答结构（90秒）**：

**第一步：核心区别**
> "最关键的区别是：自定义Hooks可以调用其他Hooks（useState、useEffect等），普通函数不可以。"

**第二步：说明原因**
> "因为React的Hooks规则要求Hooks只能在函数组件或自定义Hook中调用。React通过函数名是否use开头来判断，ESLint插件也是这样检查的。"

```jsx
// ❌ 普通函数调用Hook会报错
function formatData() {
  const [data, setData] = useState(null);  // ESLint报错
}

// ✅ 自定义Hook可以调用Hook
function useData() {
  const [data, setData] = useState(null);  // ✓
  return data;
}
```

**第三步：其他区别**
> "自定义Hooks通常返回状态和操作函数，每次调用都有独立的状态；普通函数只做计算或格式化，没有状态，适合纯函数逻辑。"

```jsx
// 普通函数：纯计算
function formatPrice(price) {
  return `¥${price.toFixed(2)}`;
}

// 自定义Hook：有状态
function usePrice(productId) {
  const [price, setPrice] = useState(0);
  // 获取价格、监听变化...
  return price;
}
```

**第四步：选择建议**
> "如果只是计算、格式化、工具函数，用普通函数；如果需要状态、副作用、或复用组件逻辑，用自定义Hooks。"

---

### 问题：自定义Hooks相比HOC有什么优势？

**回答结构（2分钟）**：

**第一步：问题对比**
> "HOC主要有三个问题：props来源不清晰、嵌套地狱、调试困难。自定义Hooks完美解决了这些问题。"

**第二步：具体说明**

**问题1：props来源不清晰**
```jsx
// HOC：mousePosition从哪来？要看HOC代码
function MyComponent({ mousePosition, user, theme }) {
  // 不知道哪个props来自哪个HOC
}

// Hooks：一目了然
function MyComponent() {
  const mousePosition = useMousePosition();
  const user = useUser();
  const theme = useTheme();
  // 数据来源清晰
}
```

**问题2：嵌套地狱**
```jsx
// HOC：3个HOC就3层嵌套
export default withRouter(
  withAuth(
    withTheme(MyComponent)
  )
);

// Hooks：扁平化
function MyComponent() {
  const router = useRouter();
  const auth = useAuth();
  const theme = useTheme();
}
```

**问题3：调试困难**
> "HOC会创建很多中间组件，React DevTools里看到一堆WithXXX，很难找到目标组件。Hooks不创建额外组件，DevTools里直接看到Hook的值。"

**第三步：补充HOC的适用场景**
> "不过HOC也不是完全没用。如果需要增强Class组件（Class不能用Hooks）、或者需要完全拦截props、或者做条件渲染，HOC还是有用的。但对于函数组件，自定义Hooks是首选。"

### 问题：什么场景适合抽取成自定义Hook？

**回答结构（90秒）**：

**第一步：判断标准**
> "有三个判断标准：逻辑是否重复、是否包含状态或副作用、是否需要复用。三个都满足就适合抽自定义Hook。"

**第二步：举例说明**

**适合抽Hook的场景：**
1. **表单输入绑定**
```jsx
function useInput(initialValue) {
  const [value, setValue] = useState(initialValue);
  const bind = {
    value,
    onChange: e => setValue(e.target.value)
  };
  return [value, bind, setValue];
}

// 使用
const [name, nameBind] = useInput('');
<input {...nameBind} />
```

2. **数据请求**
```jsx
function useFetch(url) {
  const [data, setData] = useState(null);
  const [loading, setLoading] = useState(false);
  useEffect(() => {
    setLoading(true);
    fetch(url).then(r => r.json()).then(setData).finally(() => setLoading(false));
  }, [url]);
  return { data, loading };
}
```

3. **事件监听**
```jsx
function useWindowSize() {
  const [size, setSize] = useState({ width: 0, height: 0 });
  useEffect(() => {
    const handleResize = () => setSize({ width: window.innerWidth, height: window.innerHeight });
    window.addEventListener('resize', handleResize);
    return () => window.removeEventListener('resize', handleResize);
  }, []);
  return size;
}
```

**不适合抽Hook的场景：**
> "纯计算、格式化、工具函数，用普通函数就行。比如formatDate、calculateTotal这种，没有状态也不复用React逻辑，用普通函数更简单。"

**第三步：最佳实践**
> "抽Hook前先问自己：这个逻辑是否至少在两个地方用到？是否包含状态或副作用？如果答案都是是，再抽Hook。不要过度抽象，一个地方用就没必要抽。"
