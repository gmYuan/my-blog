# React 版本迭代核心变更（17→18→19）

## 说明

本篇聚合 React 17、18、19 三个主要版本的核心变更与面试高频考点，基于掘金小册与相关源码文章。由于版本跨度大，仅提炼 **面试会问到的重点**，细节待后续章节展开。

---

## React 17 核心变更

### 1. 无新特性的过渡版本
- React 17 被称为「无新特性的版本」，主要为 **React 18 的并发特性** 做铺垫。
- 重点在 **底层变更** 而非新 API。

### 2. 事件委托变更
- **变更前（≤16）**：所有事件委托到 **`document`**。
- **变更后（17+）**：委托到 **React 挂载的根 DOM 节点**（`root`）。
- **原因**：支持 **多版本 React 共存**（如微前端场景）、避免事件冲突。

### 3. 移除事件池（Event Pooling）
- React 16 及之前，合成事件对象会被 **复用**（event pooling），异步访问 `e.target` 会失效。
- React 17 **移除事件池**，可在异步回调里直接用事件对象，不需要 `e.persist()`。

### 4. 对齐浏览器行为
- `onScroll` 不再冒泡（与原生一致）。
- `onFocus` / `onBlur` 底层改用原生 `focusin` / `focusout`。

## React 18 核心变更（Concurrent 时代）

### 1. Concurrent Mode（并发模式）
- **最重要变更**：渲染 **可中断、可恢复、可放弃**。
- **不是新 API**，而是 **底层机制**，支撑 Suspense、transition、流式 SSR 等新特性。
- **优先级调度**：高优先级任务（如 input 输入）可打断低优先级任务（如列表渲染），避免卡顿。

**核心特点：**
- **可中断**：Fiber 链表结构支持暂停与恢复。
- **可放弃**：低优先级任务若被后续更新覆盖，可直接丢弃（如频道切换例子：只关心最终频道，中间渲染可跳过）。
- **状态复用**（未完全实现）：Offscreen 组件，支持页面状态缓存与预渲染。

### 2. 新 API：createRoot
- **旧写法**：`ReactDOM.render(jsx, container)`（进入 legacy 模式）。
- **新写法**：`createRoot(container).render(jsx)`（开启 Concurrent 特性）。
- **hydrateRoot**：SSR 场景替代 `ReactDOM.hydrate`。

### 3. 自动批处理（Automatic Batching）
- **变更前**：只在 React 事件处理器里批量更新 state。
- **变更后**：**setTimeout、Promise、原生事件** 里的多个 `setState` 也会 **自动合并为一次渲染**。
- 面试常问「React 18 之前 setState 同步还是异步」的后半部分答案变了。

### 4. Suspense 与流式 SSR
- Suspense 支持 **代码分割**、**数据获取** 时的 fallback UI。
- 流式 SSR：服务端可 **分块发送 HTML**，客户端逐步 hydrate，不必等整个页面就绪。

### 5. Transitions API
- `useTransition` / `startTransition`：标记 **低优先级更新**（如搜索建议），让高优先级任务（如 input）优先响应。

### 6. 新 Hooks
- `useId`：生成稳定的唯一 ID（SSR 场景防止客户端 hydrate 不匹配）。
- `useDeferredValue`：延迟更新低优先级值。
- `useSyncExternalStore`：订阅外部数据源（如状态管理库）。
- `useInsertionEffect`：CSS-in-JS 库专用，在 DOM 变更前插入样式。

## React 19 核心变更

### 1. React Compiler（编译器优化）
- **自动优化**：无需手写 `useMemo` / `useCallback` / `memo`，编译器自动分析依赖并优化。
- 仍在实验阶段，逐步推广中。

### 2. Actions 与表单增强
- **`useActionState`**（原 `useFormState`）：处理表单提交的异步状态（pending/error/data）。
- **`useFormStatus`**：在表单子组件里读取父表单的提交状态。
- **`useOptimistic`**：乐观更新（UI 先响应，后端失败再回滚）。

### 3. `use` Hook
- **突破规则**：可以在 **条件语句** 里调用（仅限 `use`）。
- 用于 **读取 Promise 或 Context**，如 `const data = use(promise)`。

### 4. ref 简化
- **不再需要 `forwardRef`**：函数组件可直接接收 `ref` 作为 prop。
- Cleanup 函数：`ref` 回调可返回清理函数（类似 `useEffect`）。

### 5. Context 改进
- **`<Context>` 作为 Provider**：不再写 `<Context.Provider>`，直接 `<ThemeContext value={...}>`。

### 6. 文档元数据支持
- 组件里直接写 `<title>` / `<meta>` / `<link>`，React 自动提升到 `<head>`，无需第三方库。

### 7. 资源预加载 API
- `preload` / `preinit` / `prefetchDNS`：声明式预加载样式、脚本、字体等。

## 面试高频点（版本演进题）

### 问法示例
- React 17/18/19 分别有什么重要变更？
- 什么是 Concurrent Mode？为什么重要？
- React 18 的自动批处理是什么？
- `createRoot` 和 `ReactDOM.render` 有什么区别？
- React 19 的 `use` Hook 为什么能在条件里用？

### 答题策略
- **React 17**：一句话带过「事件委托改到 root、移除事件池」。
- **React 18**：重点说 **Concurrent（可中断渲染、优先级调度）**、**自动批处理**、**createRoot**、**Suspense + 流式 SSR**。
- **React 19**：**React Compiler**（自动优化）、**Actions 与表单**、**`use` Hook**、**ref 简化**。
