# 【知识点总览】React 源码中的数据结构与算法

## 说明

本篇非单一面试题，而是 **React 源码涉及的技术栈清单**（PDF 第 3 页提到）。学习源码过程中会遇到这些知识点，可作为 **学习路线图** 或 **查漏补缺清单**。

---

## React 源码中的核心知识点（按 PDF 原文）

### 数据结构
- **VDOM**（虚拟 DOM）  
- **Fiber**（React 16+ 的核心架构）  
- **单链表**  
- **循环链表**  
- **最小堆**（Scheduler 调度器中用于任务优先级）  
- **哈希表**  
- **栈与队列**

### 算法
- **深度优先搜索**（DFS）  
- **位运算**（用于 Lanes 优先级模型、状态标记等）  
- **单线程任务调度器**（Scheduler，时间切片与优先级调度）

---

## 面试可能涉及的问法

虽然不会直接问「React 用了哪些数据结构」，但这些知识点会 **嵌在具体题里**，例如：

- Fiber 是什么？为什么用链表而不是树？ → **单链表**  
- React 如何实现时间切片？ → **Scheduler、最小堆**  
- Lanes 模型怎么表示优先级？ → **位运算**  
- Reconciliation 怎么遍历 Fiber 树？ → **深度优先搜索**

---

## 使用建议

- **学源码时**：遇到不熟悉的数据结构（如 Fiber 链表、最小堆），先补基础；  
- **面试准备**：重点掌握 **Fiber、Lanes、Scheduler、VDOM Diff**，其余作为「能说上一两句」的加分项。
