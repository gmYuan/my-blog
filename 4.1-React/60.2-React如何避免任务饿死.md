# React 如何避免任务饿死

## 什么是任务饿死（Task Starvation）？

**饿死现象：**
- 低优先级任务长时间得不到执行
- 高优先级任务不断到来，低优先级任务一直被推迟
- 最终导致低优先级任务"饿死"（永远得不到执行）

**为什么会饿死？**
- 在抢占式调度器中，高优先级任务总是优先执行
- 如果高优先级任务源源不断，低优先级任务永远排不到

---

## React 的两种解决方案

React 采用 **合作式调度器**，并通过两种机制避免任务饿死：

### 1️⃣ 时间切片（Time Slicing）

**机制：**
- 划分时间段，避免某些任务长期占用主线程
- 默认时间片：**5ms**
- 即使是高优先级任务，也不能无限占用主线程

**作用：**
- 避免长时间运行的任务阻塞其它任务的执行
- 提高整体的响应性和性能
- 给低优先级任务执行的机会

**实现位置：**
- `react/scheduler` 包
- `shouldYieldToHost()` 函数判断时间片是否用完
- `workLoop` 中检查 `shouldYieldToHost()`，决定是否让出

---

### 2️⃣ Aging 机制（老化机制）

**机制：**
- 对于长时间没法得到处理的低优先级任务（即快要饿死的任务）
- **提高它们的优先级**，以此避免任务饿死

**实现位置：**
- `react/react-scheduler` 包
- `markStarvedLanesAsExpired` 函数

**具体做法：**
1. 遍历待处理的任务（遍历 lanes）
2. 检查它们是否过期
3. 如果已经过期，就认为该任务处于即将饿死的状态
4. 标记为已经过期，以强制它的完成

**核心思想：**
- 把"饿死任务"标记为过期，相当于提高优先级
- 过期任务会被强制完成，避免永久等待

---

## 两种方案的配合

| 机制 | 作用 | 目标 |
|------|------|------|
| **时间切片** | 限制单次执行时间 | 避免高优先级任务霸占主线程 |
| **Aging** | 提升等待过久任务的优先级 | 避免低优先级任务永久等待 |

**结合使用：**
- 时间切片保证了"公平性"（每个任务最多执行 5ms）
- Aging 保证了"最终执行"（等太久的任务会被提升优先级）
