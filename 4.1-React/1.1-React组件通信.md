# 【高频】React 组件通信：props、callback、children、状态提升

## 常怎么问

- 父子组件怎么传数据？子怎么通知父？  
- `children` 是什么？  
- 什么是状态提升？

## 答题要点

### 父 → 子

- 通过 **`props`** 传数据、传函数。  
- 写在组件标签 **内部的内容** 在 **`props.children`**，用于组合 UI。

**为什么组件包裹的内容都在 `children` 里？**

- **组合 > 继承**：React 鼓励用 **组件嵌套** 而非深继承来复用 UI；`children` 让父组件 **不用预知** 子组件具体会塞什么内容，实现 **插槽（Slot）** 效果（类似 Vue 的 `<slot />`）。  
- **声明式语法**：`<Card>这里是内容</Card>` 比 `<Card content="..." />` 更接近 HTML 直觉，也便于嵌套复杂结构（如多层 JSX、多个子组件）。  
- **灵活性**：`children` 可以是字符串、JSX、数组、函数等，父组件通过 `props.children` 拿到后可做包装、遍历、条件渲染等，**控制反转** 给使用者。

**一句话：** `children` 把「组件内部放什么」的决定权交给 **调用方**，让组件像容器一样可复用。

### 子 → 父

- 父组件把 **`setState` / `setXxx` 或自定义回调** 通过 props 传给子组件。  
- 子组件在事件里调用该回调并传入新值。  
- 数据放在父组件、子组件只负责触发更新，叫 **状态提升（lifting state up）**，便于多个子树共享同一数据源。

### 顺带一句（分类题）

- 面试若问「通信方式」，在答清 props/callback 后，可补：**跨层级用 Context**（见 **1.3**），**全局复杂流用状态管理**（Redux Toolkit / Zustand 等），**ref 为补充**（命令式场景，如拿子组件实例、聚焦 DOM）。

## 一句话收尾

「单向数据流：向下 props，向上 callback；共享状态往上提。」
