# createContext的默认值什么时候使用

## 说明

createContext 的参数是 context value 的默认值,当后代组件消费 context value 但是匹配不到对应 Provider 的时候,会取值这个默认值。

## 示例

```javascript
import { createContext } from 'react';

const CountContext = createContext(-999);
```

当组件使用 `useContext(CountContext)` 或其他方式消费这个 context,但是在组件树中找不到对应的 `<CountContext.Provider>` 时,就会使用默认值 `-999`。
