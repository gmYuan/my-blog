# 【高频】Context 与 useContext：跨层级如何传值？

## 当前业界常见说法

**「除了 props 怎么跨层传」**、**「主题 / 语言 / 用户信息怎么下发」** 这类题，常见答法会落到 **Context + `useContext`**。预习篇若用类组件 + `contextType` 举例，答题时仍以 **函数组件 + `useContext`** 为主即可；类组件消费 Context 多见于 **遗留代码**，不必当作当前栈的默认写法。

## 常怎么问

- 跨很多层组件传参，不想层层 props，怎么办？  
- Context 解决什么问题？`Provider` / `Consumer` / `useContext` 各自干嘛？  
- 和 Vue 的 provide / inject 有什么关系？

## 答题要点

### 解决什么问题

- **跨层级共享**：祖先提供、后代消费，避免 **props drilling**。  
- 典型：**主题、语言、当前用户、权限** 等「半全局、但不必上 Redux」的数据。

### API 三步

1. **`createContext(defaultValue)`**  
2. 上层 **`<Ctx.Provider value={...}>`**  
3. 消费：**`useContext(Ctx)`**（当前默认）。类组件可用 `static contextType = Ctx` + `this.context`，但属遗留写法，面试很少单独考。

### 和 Vue

- 思路接近 **provide / inject**：上层提供、下层注入。

## 面试常追问（略答即可）

- **`value` 引用变了会怎样？** 会触发该 Context 下的消费组件更新；大对象可用 `useMemo` 稳定引用、或 **拆多个 Context** 减无效渲染。

## 一句话收尾

「跨层共享用 Context；新项目消费端优先 `useContext`。」
