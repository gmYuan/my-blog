# Context完整指南

## 传递 props 带来的问题

传递 props 是将数据通过 UI 树显式传递到使用它的组件的好方法。

但是当你需要在组件树中深层传递参数以及需要在组件间复用相同的参数时,传递 props 就会变得很麻烦。最近的根节点父组件可能离需要数据的组件很远,状态提升到太高的层级会导致 "逐层传递 props" 的情况。

要是有一种方法可以在组件树中不需要 props 将数据"直达"到所需的组件中,那可就太好了。React 的 context 功能可以满足我们的这个心愿。

## Context 的使用场景

Context 使用场景：当祖先组件想要和后代组件快速通信。

- **主题**: 如果你的应用允许用户更改其外观（例如暗夜模式）,你可以在应用顶层放一个 context provider,并在需要调整其外观的组件中使用该 context。

- **当前账户**: 许多组件可能需要知道当前登录的用户信息。将它放到 context 中可以方便地在树中的任何位置读取它。某些应用还允许你同时操作多个账户（例如,以不同用户的身份发表评论）。在这些情况下,将 UI 的一部分包裹到具有不同账户数据的 provider 中会很方便。

- **路由**: 大多数路由解决方案在其内部使用 context 来保存当前路由。这就是每个链接"知道"它是否处于活动状态的方式。如果你创建自己的路由库,你可能也会这么做。

- **状态管理**: 随着你的应用的增长,最终在靠近应用顶部的位置可能会有很多 state。许多遥远的下层组件可能想要修改它们。通常将 reducer 与 context 搭配使用来管理复杂的状态并将其传递给深层的组件来避免过多的麻烦。

## API 用法

### 1. 创建 context

```javascript
import { createContext } from 'react';

export const LevelContext = createContext(1);
```

注意：createContext 的参数是 context value 的默认值,当后代组件消费 context value 但是匹配不到对应 Provider 的时候,会取值这个默认值。

### 2. 分发 value - Provider

### 3. 后代组件消费 value

1) useContext - 函数组件
2) ContextType - 类组件且只能消费单一 context 来源
3) Consumer - 组件

## useContext 的作用

useContext 用于在函数组件中消费 Context 的值。

### 用法示例

```javascript
import { useContext } from 'react';

function Child({ desc }) {
  const background = useContext(MyBGContext);
  const count = useContext(CountContext);
  
  return (
    <div className="border" style={{ background }}>
      <h3>{desc}</h3>
      <h5>{count}</h5>
    </div>
  );
}
```

### 说明

- useContext 是函数组件中消费 Context 的方式
- 类组件可以使用 `static contextType` 或 `<Context.Consumer>` 来消费 Context
- 当有多个 Provider 嵌套时,useContext 会取最近的那个 Provider 的值
- 当找不到匹配的 Provider 时,会使用 createContext 时的默认值

## createContext 的默认值使用时机

createContext 的参数是 context value 的默认值,当后代组件消费 context value 但是匹配不到对应 Provider 的时候,会取值这个默认值。

### 示例

```javascript
import { createContext } from 'react';

const CountContext = createContext(-999);
```

当组件使用 `useContext(CountContext)` 或其他方式消费这个 context,但是在组件树中找不到对应的 `<CountContext.Provider>` 时,就会使用默认值 `-999`。

## 完整代码示例

```javascript
import {
  Component,
  useReducer,
  useState,
  createContext,
  useContext,
} from "react";

const CountContext = createContext(-999);
const MyBGContext = createContext("red");

export default function ContextPage2() {
  const [count, setCount] = useReducer((x) => x + 1, 0);
  const [bgColor, setBgColor] = useState("red");
  
  return (
    <div className="border">
      <button onClick={() => setCount()}>{count}</button>
      <button onClick={() => setBgColor(getRandomColor())}>
        change theme color
      </button>
      
      {/* 没有匹配的 Provider,取值默认值 */}
      <Child desc="没有匹配的 Provider" />
      
      {/* 匹配到多个 Provider,取最近的那个 */}
      <CountContext.Provider value={999}>
        <CountContext.Provider value={count}>
          <Child desc="匹配到多个 Provider" />
          <MyBGContext.Provider value={bgColor}>
            {/* 不同 Context Provider 嵌套 */}
            <Child desc="Context Provider" />
          </MyBGContext.Provider>
        </CountContext.Provider>
      </CountContext.Provider>
    </div>
  );
}

function Child({ desc }) {
  const background = useContext(MyBGContext);
  const count = useContext(CountContext);
  
  return (
    <div className="border" style={{ background }}>
      <h3>{desc}</h3>
      <h5>{count}</h5>
      <CountContext.Consumer>{(ctx) => <h5>{ctx}</h5>}</CountContext.Consumer>
      <ClassComponent />
    </div>
  );
}

class ClassComponent extends Component {
  static contextType = CountContext;
  
  render() {
    return (
      <div className="border">
        类组件
        <p>:{this.context}</p>
      </div>
    );
  }
}

function getRandomColor(props) {
  return "#" + Math.floor(Math.random() * Math.pow(10, 6));
}
```
