# React组件key的作用是什么

## 作用

key 值标记了节点在当前层级下的唯一性。

在协调阶段，组件复用的前提是必须同时满足三个条件：
1. 同一层级下
2. 同一类型
3. 同一个 key 值

所以我们要尽量保证这三者的稳定性。

## 常见错误

### 错误写法
- `key=Math.random()` - 每次都生成新的 key，破坏了稳定性

### 不规范写法
- `key=index` - 尽量不要取值 index

## 为什么不推荐使用 index 作为 key

因为在同一层级下，多个循环的 index 容易重复。

并且如果涉及节点的增加、删除、移动，那么 key 的稳定性将会被破坏，节点就会出现混乱现象。
