## 📖 版本演进概览（了解即可）

### React 16.3（Fiber时代开启）
- **定位**: Fiber架构正式引入
- **核心变更**: 
  - 引入Fiber架构（可中断渲染的基础）
  - 新的生命周期API（getDerivedStateFromProps等）
- **面试价值**: 中等，可能问"Fiber什么时候引入的？"

### React 16.8（Hooks革命）⭐
- **定位**: Hooks正式发布，改变React写法
- **核心变更**: 
  - useState、useEffect等Hooks API
  - 函数组件可以有状态
  - 逐步替代Class组件
- **面试价值**: 高，常问"Hooks哪个版本引入？答案：16.8"

### React 17（过渡版本）
- **定位**: 无新特性的过渡版本，为React 18铺路
- **核心变更**: 事件委托从document改到root节点
- **面试价值**: 低，一般不问

### React 18（并发时代）
- **定位**: 开启并发特性的里程碑版本
- **核心变更**: 
  - Concurrent Mode（可中断渲染）→ 见6.2
  - 自动批处理 → 见6.3
  - createRoot → 见6.4
  - Suspense + 流式SSR
  - useTransition / startTransition
- **面试价值**: 极高，必考

### React 19（最新版本）
- **定位**: 编译器优化 + DX改进
- **核心变更**:
  - React Compiler（自动优化）
  - Actions与表单增强
  - use Hook（可在条件中调用）
  - ref简化（不需要forwardRef）
- **面试价值**: 低（太新，实际项目少用）

## 面试快速回答

### Q: React主要版本有哪些重要变更？

**回答（1-2分钟）**：
> "React有几个关键版本：16.3引入Fiber架构，16.8引入Hooks让函数组件可以有状态。React 18是最重要的版本，Concurrent Mode让渲染可以被高优先级任务打断，比如用户输入可以打断列表渲染，避免卡顿。自动批处理让所有场景下的setState都能合并更新。React 17主要是底层改动，React 19引入了编译器优化。"

### Q: 你们项目用的React什么版本？

**回答策略**：
- 如果用17：提一下考虑升级18获得性能提升
- 如果用18：说说concurrent特性带来的好处
- 如果用16：说说升级的计划（不要说"太老了"）
