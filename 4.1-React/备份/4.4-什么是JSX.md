# 什么是JSX？

## 定义

**JSX（JavaScript XML）**是一种JavaScript的语法扩展，它允许开发者用类似HTML的语法来编写React组件。

JSX看起来像HTML，但本质上是JavaScript。

---

## 基本语法

```jsx
// JSX语法
const element = <h1>Hello, World!</h1>;

// JSX可以包含表达式
const name = 'John';
const element = <h1>Hello, {name}!</h1>;

// JSX可以嵌套
const element = (
  <div>
    <h1>Title</h1>
    <p>This is a paragraph</p>
  </div>
);
```

---

## JSX的本质

### 1️⃣ JSX是语法糖

JSX实际上是`React.createElement()`的语法糖：

```jsx
// 编写JSX
<h1 className="title">Hello</h1>

// 被Babel编译为
React.createElement('h1', { className: 'title' }, 'Hello')

// React.createElement返回一个对象（ReactElement）
{
  type: 'h1',
  props: {
    className: 'title',
    children: 'Hello'
  },
  key: null,
  ref: null,
  // ...
}
```

### 2️⃣ JSX必须编译

浏览器无法直接解析JSX，需要通过工具（如Babel）编译成标准JavaScript：

```jsx
// 源代码（JSX）
function App() {
  return <div>Hello</div>;
}

// 编译后（JavaScript）- React 17之前
function App() {
  return React.createElement('div', null, 'Hello');
}

// 编译后（JavaScript）- React 17+
import { jsx as _jsx } from 'react/jsx-runtime';

function App() {
  return _jsx('div', { children: 'Hello' });
}
```

---

## React 17的JSX转换变化

### React 17之前

- JSX必须转换为`React.createElement(...)`调用
- 必须在文件顶部导入React：`import React from 'react';`
- 即使不显式使用React，也必须导入

### React 17+

- JSX转换不再使用`React.createElement`
- 自动从React的package中引入新的入口函数（`jsx`、`jsxs`）
- 不再需要导入React（除非使用Hooks等API）

```jsx
// React 17+ 不需要导入React
// import React from 'react';  // ❌ 不再需要

function App() {
  return <div>Hello</div>;  // ✅ 可以直接使用JSX
}
```

## JSX与HTML的区别

| 特性 | HTML | JSX |
|-----|------|-----|
| **class属性** | `class="foo"` | `className="foo"` |
| **for属性** | `<label for="input">` | `<label htmlFor="input">` |
| **style属性** | `style="color: red"` | `style={{ color: 'red' }}` |
| **自闭合标签** | `<img>` 或 `<img />` | 必须 `<img />` |
| **注释** | `<!-- comment -->` | `{/* comment */}` |
| **JavaScript表达式** | 不支持 | `{expression}` |
| **事件命名** | `onclick` | `onClick`（驼峰命名） |

## JSX的优势

### 1️⃣ 直观易读

```jsx
// JSX - 直观，类似HTML
return (
  <div className="container">
    <h1>Title</h1>
    <p>Content</p>
  </div>
);

// 不用JSX - 嵌套复杂，难以阅读
return React.createElement(
  'div',
  { className: 'container' },
  React.createElement('h1', null, 'Title'),
  React.createElement('p', null, 'Content')
);
```

### 2️⃣ 开发效率高

- 类似HTML的语法，学习成本低
- 编辑器支持好（语法高亮、自动补全）
- 快速编写UI结构

### 3️⃣ 类型安全

编译过程中能发现错误：
```jsx
// 编译时错误：JSX标签必须闭合
<div>  // ❌ 编译失败

// 编译时错误：属性名错误
<div classname="foo">  // ⚠️ 应该是 className
```

### 4️⃣ 执行效率高

JSX编译为JavaScript代码后进行了优化，执行更快。

---

## JSX与Vue模板的对比

| 对比项 | React JSX | Vue Template |
|-------|----------|--------------|
| **语法** | JavaScript扩展 | HTML扩展 |
| **本质** | 本质就是JS | 需要编译为render函数 |
| **出现时间** | React一开始就有 | Vue演进过程中才出现 |
| **编译复杂度** | 简单直接 | 复杂（字符串→AST→JS函数） |
| **灵活性** | 高（完全JS能力） | 中（受模板语法限制） |
| **学习曲线** | 需要懂JS | 更接近HTML，易上手 |

## 常见使用场景

### 1. 动态内容

```jsx
const name = 'John';
const element = <h1>Hello, {name}!</h1>;
```

### 2. 条件渲染

```jsx
const isLoggedIn = true;
return (
  <div>
    {isLoggedIn ? <UserProfile /> : <LoginButton />}
  </div>
);
```

### 3. 列表渲染

```jsx
const items = ['Apple', 'Banana', 'Orange'];
return (
  <ul>
    {items.map(item => <li key={item}>{item}</li>)}
  </ul>
);
```

### 4. 事件处理

```jsx
function handleClick() {
  alert('Clicked!');
}

return <button onClick={handleClick}>Click Me</button>;
```

## 注意事项

1. **JSX必须有唯一根元素**
   ```jsx
   // ❌ 错误
   return (
     <div>First</div>
     <div>Second</div>
   );
   
   // ✅ 正确 - 使用Fragment
   return (
     <>
       <div>First</div>
       <div>Second</div>
     </>
   );
   ```

2. **JSX中的JavaScript表达式必须有返回值**
   ```jsx
   // ❌ 不能使用语句
   <div>{if (true) { return 'yes'; }}</div>
   
   // ✅ 使用表达式
   <div>{true ? 'yes' : 'no'}</div>
   ```

3. **组件名必须大写**
   ```jsx
   // ❌ 小写会被认为是HTML标签
   <myComponent />
   
   // ✅ 大写表示React组件
   <MyComponent />
   ```
