## 第一部分：顶层鸟瞰

### 背景问题
这批截图在解决一个问题：**如何在 React Monorepo 项目中配置 Rollup 打包工具，实现多包构建、TypeScript 编译、以及自动生成 package.json 等功能。**

### 宏观图

```
React-P1/
├── packages/
│   ├── react/
│   │   ├── src/
│   │   │   └── jsx.ts                # ✅ JSX 实现源码
│   │   ├── index.ts                   # ✅ react 包入口
│   │   └── package.json              # ✅ react 包配置
│   └── shared/
│       ├── ReactSymbols.ts           # ✅ Symbol 常量
│       ├── ReactTypes.ts             # ✅ TypeScript 类型
│       └── package.json
├── scripts/
│   └── rollup/
│       ├── utils.js                  # ✅ Rollup 工具函数
│       └── react.config.js           # ✅ react 包构建配置
├── dist/                              # ✅ 构建输出目录
│   └── node_modules/
│       └── react/
│           ├── index.js              # ✅ 构建后的主入口
│           ├── jsx-runtime.js        # ✅ 生产环境 runtime
│           ├── jsx-dev-runtime.js    # ✅ 开发环境 runtime
│           └── package.json          # ✅ 自动生成的配置
├── package.json                       # ✅ 根配置（含构建脚本）
└── pnpm-workspace.yaml               # ✅ Workspace 配置
```

**这个实现方案解决的问题：**

| 模块 | 解决什么问题 |
|------|-------------|
| **utils.js** | 提供路径解析、读取 package.json、获取基础插件等工具函数 |
| **react.config.js** | 配置 react 包的构建规则，包括入口、输出、插件等 |
| **Rollup 插件** | TypeScript 编译、CommonJS 转换、生成 package.json 等 |
| **多入口构建** | 同时构建 react 主包和 jsx-runtime |
| **dist 目录** | 统一输出所有构建产物，便于发布和测试 |

**Rollup 打包的本质：**

源码（TypeScript）→ Rollup 编译 → 输出（JavaScript + package.json）→ dist/node_modules

```
输入：packages/react/src/jsx.ts
↓
Rollup 处理：
  - TypeScript 编译
  - 代码转换（ES6 → CommonJS/ESM）
  - 插件处理
↓
输出：dist/node_modules/react/
  - index.js（主入口）
  - jsx-runtime.js（生产环境）
  - jsx-dev-runtime.js（开发环境）
  - package.json（自动生成）
```

### 步骤 0：修复命名冲突（重要）

**问题：** `jsx.ts` 中有一个函数叫 `ReactElement`，而 `ReactTypes.ts` 中也有一个接口叫 `ReactElement`，会导致命名冲突。

**解决方案：** 将 `ReactTypes.ts` 中的接口 `ReactElement` 改名为 `ReactElementType`。

**修改文件：** `packages/shared/ReactTypes.ts`

```typescript
// 基础类型定义（暂时用 any 占位，后续细化）
export type Type = any;
export type Key = any;
export type Ref = any;
export type Props = any;
export type ElementType = any;

// ✅ 接口名改为 ReactElementType（避免和 jsx.ts 中的函数 ReactElement 冲突）
export interface ReactElementType {
  $$typeof: symbol | number;
  type: ElementType;
  key: Key;
  props: Props;
  ref: Ref;
  __mark: string;
}
```

**对应修改 jsx.ts 中的导入：**

```typescript
// packages/react/src/jsx.ts
import { REACT_ELEMENT_TYPE } from 'shared/ReactSymbols';
import {
  Type,
  Key,
  Ref,
  Props,
  ReactElementType,  // ← 导入改为 ReactElementType
  ElementType
} from 'shared/ReactTypes';

// ReactElement 函数（注意：这是函数名）
const ReactElement = function (
  type: Type,
  key: Key,
  ref: Ref,
  props: Props
): ReactElementType {  // ← 返回类型是 ReactElementType
  const element = {
    $$typeof: REACT_ELEMENT_TYPE,
    type,
    key,
    ref,
    props,
    __mark: 'Ygm'
  };
  return element;
};
```

**命名解释：**

| 名称 | 类型 | 位置 | 作用 |
|------|------|------|------|
| `ReactElementType` | TypeScript 接口 | `shared/ReactTypes.ts` | 定义 React 元素的数据结构 |
| `ReactElement` | JavaScript 函数 | `packages/react/src/jsx.ts` | 创建 React 元素的工厂函数 |

这样就避免了函数名和类型名冲突的问题。

### 步骤 1：创建 Rollup 工具脚本（utils.js）

**路径：** `scripts/rollup/utils.js`

```javascript
import path from 'path';
import fs from 'fs';
import ts from 'rollup-plugin-typescript2';
import cjs from '@rollup/plugin-commonjs';

// packages 目录的路径
const pkgPath = path.resolve(__dirname, '../../packages');
// 构建产物输出目录
const distPath = path.resolve(__dirname, '../../dist/node_modules');

/**
 * 解析包的路径
 * @param {string} pkgName - 包名（如 'react'）
 * @param {boolean} isDist - 是否是 dist 路径
 */
export function resolvePkgPath(pkgName, isDist) {
  if (isDist) {
    return `${distPath}/${pkgName}`;  // dist/node_modules/react
  }
  return `${pkgPath}/${pkgName}`;     // packages/react
}

/**
 * 读取包的 package.json
 * @param {string} pkgName - 包名
 */
export function getPackageJSON(pkgName) {
  // ...包路径
  const path = `${resolvePkgPath(pkgName)}/package.json`;
  const str = fs.readFileSync(path, { encoding: 'utf-8' });
  return JSON.parse(str);
}

/**
 * 获取基础 Rollup 插件配置
 * @param {Object} options - typescript 配置选项
 */
export function getBaseRollupPlugins({ typescript = {} } = {}) {
  return [cjs(), ts(typescript)];
}
```

**关键点：**

| 函数 | 作用 |
|------|------|
| `resolvePkgPath` | 解析包的源码路径或输出路径 |
| `getPackageJSON` | 读取并解析包的 package.json |
| `getBaseRollupPlugins` | 获取基础插件（CommonJS + TypeScript） |

### 步骤 2：配置 React 包构建（react.config.js）

**路径：** `scripts/rollup/react.config.js`

```javascript
import { getPackageJSON, resolvePkgPath, getBaseRollupPlugins } from './utils';
import generatePackageJson from 'rollup-plugin-generate-package-json';

// ============================================================
// 第一步：读取 react 包的 package.json
// ============================================================
const { name, module } = getPackageJSON('react');  // ← 获取包名和入口模块

// react包的路径（packages/react）
const pkgPath = resolvePkgPath(name);
// react产物路径（dist/node_modules/react）
const pkgDistPath = resolvePkgPath(name, true);

export default [
  // ============================================================
  // react 主包构建配置
  // ============================================================
  {
    input: `${pkgPath}/${module}`,                  // ← 输入：packages/react/index.ts
    output: {
      file: `${pkgDistPath}/index.js`,              // ← 输出：dist/node_modules/react/index.js
      name: 'index.js',
      format: 'umd'                                 // ← UMD 格式（兼容多种模块系统）
    },
    plugins: [
      ...getBaseRollupPlugins(),                    // ← TypeScript 编译 + CommonJS 转换
      generatePackageJson({                         // ← 自动生成 package.json
        inputFolder: pkgPath,
        outputFolder: pkgDistPath,
        baseContents: ({ name, description, version }) => ({
          name,
          description,
          version,
          main: 'index.js'                          // ← 指定主入口
        })
      })
    ]
  },
  // ============================================================
  // jsx-runtime 构建配置（React 17+ 新 JSX 转换）
  // ============================================================
  {
    input: `${pkgPath}/src/jsx.ts`,                 // ← 输入：packages/react/src/jsx.ts
    output: [
      // jsx-runtime（生产环境）
      {
        file: `${pkgDistPath}/jsx-runtime.js`,
        name: 'jsx-runtime.js',
        format: 'umd'
      },
      // jsx-dev-runtime（开发环境）
      {
        file: `${pkgDistPath}/jsx-dev-runtime.js`,
        name: 'jsx-dev-runtime.js',
        format: 'umd'
      }
    ],
    plugins: getBaseRollupPlugins()                 // ← TypeScript 编译 + CommonJS 转换
  }
];
```

**关键配置说明：**

| 配置项 | 说明 |
|-------|------|
| `input` | 构建入口文件 |
| `output.file` | 输出文件路径 |
| `output.name` | 构建后的模块名称 |
| `output.format` | 输出格式（umd 兼容多种模块系统） |
| `plugins` | 使用的插件列表 |

**为什么需要两个构建配置？**

| 构建目标 | 输入文件 | 输出文件 | 特殊处理 |
|---------|---------|---------|---------|
| **react 主包** | `packages/react/index.ts` | `dist/node_modules/react/index.js` | 自动生成 package.json |
| **jsx-runtime** | `packages/react/src/jsx.ts` | `jsx-runtime.js`<br>`jsx-dev-runtime.js` | 提供 React 17+ 的新 JSX 转换运行时 |

**为什么需要 jsx-runtime？**

React 17 引入了新的 JSX 转换机制：
- **旧方式（React 17 之前）**：JSX 编译为 `React.createElement()` 调用
- **新方式（React 17+）**：JSX 编译为 `jsx()` 或 `jsxDEV()` 调用
- 需要单独打包 jsx-runtime 以支持新的转换方式

### 步骤 4：执行构建

**构建脚本说明：**

```json
{
  "scripts": {
    "build:dev": "rimraf dist && rollup --bundleConfigAsCjs --config scripts/rollup/react.config.js"

}
```

**参数说明：**

**执行构建：**

### 步骤 6：实际项目中的配置差异说明

在实际运行过程中，由于工具链版本升级和环境差异，当前项目的实现与文档中的基础配置有一些必要的调整。以下是具体差异和原因说明：

#### 差异 1：`utils.js` 中的插件配置

**文档中的实现（基础版本）：**

```javascript
export function getBaseRollupPlugins({ typescript = {} } = {}) {
  return [cjs(), ts(typescript)];
}
```

**实际项目中的实现（兼容版本）：**

```javascript
export function getBaseRollupPlugins({ typescript = {} } = {}) {
  return [
    ts({
      tsconfig: path.resolve(__dirname, '../../tsconfig.json'),
      include: ['packages/**/*.ts', 'packages/**/*.tsx'],
      exclude: ['node_modules/**'],
      ...typescript
    }),
    cjs()
  ];
}
```

**差异说明：**

| 配置项 | 文档版本 | 实际版本 | 原因 |
|--------|----------|----------|------|
| **插件顺序** | `cjs()` 在前，`ts()` 在后 | `ts()` 在前，`cjs()` 在后 | Rollup 4.x 需要先完成 TS 编译，再进行模块转换 |
| **tsconfig 路径** | 未显式指定 | 显式指定绝对路径 | 确保 monorepo 环境下准确读取配置 |
| **include/exclude** | 未指定 | 明确指定扫描范围 | 避免 Rollup 解析到不必要的文件，提升编译稳定性 |

**为什么需要调整？**

在使用 Rollup 4.x + TypeScript 5.x 的环境下，如果直接使用文档中的简单配置：
1. Rollup 会先尝试用自己的解析器处理 `.ts` 文件
2. 遇到 TypeScript 类型语法（如 `type: Type`）时会报错 `Expected ',', got ':'`
3. 导致构建失败，提示 `Could not resolve "./src/jsx"`

通过将 `ts()` 插件前置并添加明确配置，可以确保：
- TypeScript 文件在被 Rollup 解析前就已经转译成纯 JavaScript
- 减少路径解析歧义，提升构建稳定性
